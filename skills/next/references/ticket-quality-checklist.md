# 티켓 품질 체크리스트

Plan 수립 시 현재 티켓이 아래 기준을 충족하는지 확인합니다.
충족하지 않으면 → 작업 시작 전에 사용자에게 티켓 보완을 요청합니다.

## 필수 기준

- [ ] **AC가 2개 이상인가?** - 단일 AC는 너무 모호할 가능성이 높음
- [ ] **AC가 검증 가능한가?** - "잘 동작" (X) → "테스트 PASS" (O)
- [ ] **AC가 구체적인가?** - "로그인 구현" (X) → "POST /api/auth/login 엔드포인트" (O)
- [ ] **2시간 내 완료 가능한 크기인가?** - 아니면 티켓 분리 필요

## 경고 신호 (BLOCKED 가능성 높음)

| 신호                        | 대응                                 |
| --------------------------- | ------------------------------------ |
| AC에 "등"이나 "기타"가 포함 | 범위 불명확 → 사용자에게 명확화 요청 |
| 외부 API 연동이 AC에 포함   | 인증 정보/환경 확인 필요             |
| DB 스키마 변경 필요         | migration 계획 먼저 수립             |
| 기존 코드 대량 수정 필요    | 티켓 분리 제안                       |
