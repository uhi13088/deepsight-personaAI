# status - 프로젝트 상태 요약

TASK.md를 읽고 현재 프로젝트 상태를 요약합니다.

## 트리거

- 사용자가 "상태", "status", "진행률" 등을 요청할 때

## 실행 순서

### Step 1: 데이터 수집

1. `TASK.md` 읽기 - 전체 티켓 상태
2. 마지막 커밋 확인 (`git log -1`)
3. 최근 테스트 결과 확인 (가능 시)

### Step 2: 상태 요약 생성

## 출력 형식

```
## 프로젝트 상태

### 진행률
[████████░░] 80% (4/5 완료)

### IN_PROGRESS
- T3: 티켓제목
  - AC 진행: 2/3 완료

### QUEUE (대기: n개)
1. T4: 티켓제목
2. T5: 티켓제목

### DONE (완료: n개)
- T1: 티켓제목 (01-15)
- T2: 티켓제목 (01-15)

### BLOCKED (막힘: n개)
- T6: 티켓제목
  - 원인: ...
  - 필요: ...

### 권장 액션
- (다음에 할 일 제안)
```

## 추가 확인

- 마지막 커밋 시간
- 테스트 상태 (최근 실행 결과)

## 성공 기준

- TASK.md의 모든 섹션이 정확히 반영됨
- 진행률 계산이 올바름
