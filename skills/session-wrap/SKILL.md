# session-wrap - 세션 마무리

세션 종료 전 작업 상태를 정리하고, 다음 세션을 위한 핸드오프 기록을 생성합니다.

## 트리거

- 사용자가 "마무리", "정리", "세션 끝", "session-wrap", "wrap" 등을 요청할 때
- 긴 작업을 마치고 세션을 종료하기 전

## 실행 순서

### Step 1: 세션 변경사항 수집

1. `git diff --stat` — 이번 세션에서 변경된 파일 목록
2. `git log --oneline -10` — 최근 커밋 히스토리
3. `git status` — 커밋되지 않은 변경사항 확인

### Step 2: TASK.md 상태 확인

1. `TASK.md` (또는 `tasks/TASK.md`) 읽기
2. 확인 항목:
   - 현재 IN PROGRESS 티켓이 실제 완료되었는데 DONE으로 이동하지 않았는지?
   - 작업 중 발견된 새 이슈가 BACKLOG에 등록되었는지?
   - BLOCKED 항목이 해소되었는지?
3. 불일치 발견 시 → 사용자에게 TASK.md 업데이트 제안

### Step 3: lessons.md 교차 확인

1. `tasks/lessons.md` 읽기
2. 이번 세션에서 사용자로부터 수정 지시를 받은 내용이 있었는지 대화 히스토리 검토
3. 기록되지 않은 교훈이 있으면 → 추가 제안 (카테고리별)

### Step 4: 미커밋 변경 확인

1. `git status`에서 staged/unstaged 변경이 있는지 확인
2. 있으면 → 커밋 또는 스태시 제안
3. 없으면 → "모든 변경사항이 커밋되었습니다" 확인

### Step 5: 핸드오프 기록 생성

수집된 정보를 바탕으로 다음 세션을 위한 핸드오프 요약을 출력합니다.

## 출력 형식

```
## Session Wrap 결과

### 이번 세션 요약
- 변경 파일: N개
- 커밋: M개
- 작업 티켓: T# (상태)

### TASK.md 상태
- [x] IN PROGRESS 티켓과 실제 상태 일치
- [ ] 새 이슈 등록 필요: (설명)

### lessons.md
- [x] 이번 세션 교훈 기록 완료
- [ ] 미기록 교훈: (내용)

### 미커밋 변경
- (없음 / 있음 → 제안)

### 다음 세션 이어갈 작업
- (TASK.md 기준 다음 우선순위 티켓)
- (주의사항 또는 선행 조건)
```

## 실패 시

- TASK.md를 읽을 수 없으면 → Step 2 건너뛰고 나머지 진행
- git repo가 아니면 → Step 1, 4 건너뛰고 나머지 진행

## 성공 기준

- 미커밋 변경사항 없음 (커밋 또는 스태시 완료)
- TASK.md와 실제 작업 상태 일치
- 이번 세션의 교훈이 lessons.md에 기록됨
- 다음 세션을 위한 핸드오프 요약 출력
