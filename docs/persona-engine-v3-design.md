## 1. 개요

### 1.1 문서의 목적

이 문서는 DeepSight 페르소나 생성 엔진의 전면적 재구성을 위한 설계서이다. 기존의 단일 6D 벡터 시스템을 **멀티벡터 다차원 구조**로 재정의하고, 여기에 **비정량적 요소(서사, 맥락, 심리)** 를 더한 뒤, 이 두 체계 사이의 **관계를 구조화**하는 것이 핵심 목표이다.

### 1.2 설계 철학

기존 시스템에서 페르소나란 "6개의 수치로 정의된 콘텐츠 평가 성향"에 불과했다. 이 접근은 매칭 알고리즘에는 효과적이었지만, 페르소나를 하나의 **입체적인 인격**으로 구현하기에는 근본적인 한계가 있었다.

v3.0이 지향하는 페르소나는 다음과 같다:

**"입력된 값대로만 반응하는 기계"가 아니라, "과거(서사)에 기반하여 현재 상황(벡터)에 따라 유동적으로 반응하는 입체적 존재"**

이를 위해 시나리오 작법론, 심리학, 서사 이론에서 검증된 모델들을 차용하여, 페르소나의 인격을 체계적으로 구축한다.

### 1.3 이론적 기반

이 설계서는 다음의 이론과 저작을 기반으로 한다.

**린다 시거 (Linda Seger) — 캐릭터 3층 구조**

시거는 캐릭터를 구축할 때 **생리적(Physiology)**, **사회적(Sociology)**, **심리적(Psychology)** 이라는 세 가지 차원을 설정해야 한다고 강조한다. 또한 캐릭터의 **일관성(Consistency)** 과 동시에 이를 깨뜨리는 **역설(Paradox)** — 즉 모순적인 행동이나 습관 — 이 캐릭터를 입체적으로 만든다고 설명한다. 이 이론은 Layer 1(사회적 차원)의 설계, 비정량적 서사 구성, 그리고 모순(Paradox) 구현 로직의 근거가 된다.

**로버트 맥키 (Robert McKee) — 압박과 본성**

맥키는 **"진정한 캐릭터는 압박(Pressure) 속에서 드러난다"** 고 했다. 성격 묘사(Characterization) — 외모, 말투, 직업 등 겉으로 보이는 특성 — 밑에는 진짜 **본성(True Character)** 이 숨어 있으며, 이는 위기와 선택의 순간에만 드러난다. 또한 맥키는 캐릭터의 **무의식적 욕망(Unconscious Desire)** 이 의식적 목표와 충돌할 때 가장 깊은 드라마가 만들어진다고 설명한다. 이 이론은 Pressure Coefficient(압박 계수), 동적 레이어 블렌딩, Layer 3(서사적 욕망)의 근거가 된다.

**Big Five 성격 모델 (OCEAN)**

심리학에서 가장 널리 검증된 성격 모델인 Big Five는 인간의 기질을 **개방성(Openness)**, **성실성(Conscientiousness)**, **외향성(Extraversion)**, **우호성(Agreeableness)**, **신경증(Neuroticism)** 의 5가지 차원으로 설명한다. 이 모델은 상황이 변해도 잘 바뀌지 않는 **고유한 기질**을 표현하는 데 적합하며, Layer 2(본원적 기질 벡터)의 차원 구성 근거가 된다.

**조셉 캠벨 (Joseph Campbell) — 영웅의 여정**

캠벨의 **단일 신화(Monomyth)** 에 따르면, 모든 서사의 주인공은 '일상 세계'에서 출발하여 '시련'을 겪고, '변화/성장'을 이루어 '귀환'하는 여정을 거친다. 이 구조는 페르소나가 시간이 지남에 따라 성숙해지는 **성장 아크(Growth Arc)** 를 구현하는 데 활용된다. Layer 3의 `growthArc` 차원이 이에 해당한다.

**폴 리쾨르 (Paul Ricoeur) — 서사 정체성과 이중 정체성**

리쾨르는 정체성을 두 가지로 구분한다. 시간이 흘러도 변하지 않는 **동일성(Idem)** 과, 이야기를 통해 끊임없이 재구성되는 **자기성(Ipse)**. 또한 리쾨르는 정체성이 고정된 실체가 아니라 **"스스로에 대해 이야기하는 방식"** 에 의해 형성된다고 본다. 그리고 **타자성(Alterity)** — 타인과의 관계 속에서 정체성이 구성된다는 개념 — 도 제시한다. 이 이론은 Layer 2(불변하는 기질 = Idem)와 Layer 3(변화하는 서사 = Ipse)의 분리, 그리고 유저와의 관계에 따른 동적 벡터 조정의 근거가 된다.

**칼 융 (Carl Jung) — 원형 이론**

융의 집단 무의식과 **원형(Archetype)** 개념 — 영웅, 현자, 장난꾸러기, 그림자 등 — 은 페르소나가 사용자와 맺는 근원적인 관계 유형을 정의하는 데 활용된다. 또한 페르소나가 속한 세대와 문화의 **시대정신(Zeitgeist)** 을 구현하는 데도 적용된다.

## 2. 기존 시스템의 한계와 재설계 동기

### 2.1 v2.0 시스템의 구조

기존 DeepSight v2.0은 **단일 6D 벡터**로 페르소나의 콘텐츠 평가 성향을 표현했다.

```
기존 6D 벡터: [depth, lens, stance, scope, taste, purpose]
각 차원: 0.0 ~ 1.0
```

이 벡터는 사용자의 6D 벡터와 **코사인 유사도**를 계산하여 매칭하는 데 사용되었다. 매칭 알고리즘 자체는 효과적이었지만, 다음과 같은 근본적 한계가 존재했다.

### 2.2 한계 분석

**한계 1: 벡터는 "무엇을, 얼마나"만 표현한다**

`Stance=0.9`라는 수치는 "이 페르소나가 비판적이다"라는 사실은 알려주지만, **왜** 비판적인지(과거에 어떤 경험을 했기에?), 비판할 때 **구체적으로 어떤 식인지**(냉소적으로? 건설적으로? 분노에 차서?)는 설명하지 못한다.

**한계 2: 모든 성향이 하나의 평면에 존재한다**

현실의 인간은 **겉모습(사회적 가면)** 과 **속마음(본성)** 이 다를 수 있다. 겉으로는 까칠하게 비평하지만 속으로는 다른 사람의 감정을 깊이 배려하는 사람, 사교적으로 보이지만 혼자 있고 싶어하는 사람 — 이런 **모순과 괴리**가 캐릭터에 입체감을 부여한다. 단일 벡터에서는 `Stance=0.9`이면서 동시에 "공감적"일 수 없다. 가면과 본성을 분리할 방법이 없다.

**한계 3: 시간 축이 없다**

페르소나의 벡터값은 한 번 설정되면 변하지 않는 정적(static) 상태였다. 현실의 사람은 경험을 통해 성장하고, 가치관이 변화하며, 과거의 상처를 극복하기도 한다. 기존 시스템에서는 이런 **시간에 따른 성격 진화**를 표현할 수 없었다.

**한계 4: 동적 반응이 불가능하다**

같은 페르소나가 같은 질문을 받아도, **누가** 묻느냐(친절한 사람 vs 무례한 사람), **언제** 묻느냐(한낮 vs 새벽 2시), **어떤 맥락에서** 묻느냐(평온한 상황 vs 논쟁 중)에 따라 반응이 달라져야 자연스럽다. 정적 벡터만으로는 이런 맥락 민감성(Context Sensitivity)을 구현할 수 없었다.

### 2.3 v3.0의 핵심 변경 사항

| 영역 | v2.0 (기존) | v3.0 (신규) |
|---|---|---|
| 벡터 구조 | 단일 6D 벡터 | 3-Layer 16D 벡터 (7+5+4) |
| 성격 모델 | 평면적 성향 수치 | 가면(Surface) / 기질(Core) / 욕망(Shadow) 분리 |
| 비정량적 요소 | 단순 텍스트 필드 (`description`, `background`) | 구조화된 4차원 메타데이터 (서사/압박/습관/시대정신) |
| 동적 행동 | 없음 (정적 벡터) | 압박 계수(Pressure Coefficient) 기반 레이어 블렌딩 |
| 성장 | 불가능 (고정 벡터) | Growth Arc 차원으로 시간축 진화 구현 |
| 모순/입체성 | 표현 불가 | Paradox Score로 레이어 간 모순 자동 측정 |
| 맥락 반응 | 없음 | 상황, 관계, 환경에 따른 실시간 벡터 조정 |

### 2.4 하위호환성

재설계는 기존 시스템을 파괴하지 않는다.

- **Layer 1**은 기존 6D 벡터의 **상위호환**이다. 기존 6개 차원을 그대로 포함하며, `sociability` 1개 차원만 추가된다.
- 기존 매칭 알고리즘은 **Layer 1만으로 그대로 동작 가능**하다.
- Layer 2와 Layer 3는 콘텐츠 생성, 대화, PersonaWorld 자율활동 등 **새로운 기능**에서 활용된다.
- 기존 `PersonaVector` 테이블의 데이터는 Layer 1으로 **무손실 마이그레이션**이 가능하다.

## 3. Phase 1: 3-Layer Orthogonal Vector System (16D)

### 3.1 아키텍처 개요

인간의 인격을 **사회적 가면(Surface)**, **타고난 기질(Core)**, **숨겨진 욕망(Shadow)** 이라는 3가지 축으로 분리하여, 각각을 독립된 다차원 벡터로 정의한다. "Orthogonal"이라 명명한 이유는 이 세 레이어가 서로 **직교(독립적)** 하기 때문이다 — 사회적 모습, 내면의 기질, 서사적 욕망은 각각 독립적으로 설정할 수 있으며, 이 독립성 덕분에 조합의 폭이 폭발적으로 늘어난다.

### 3.2 Layer 1: 사회적 가면 벡터 (The Persona Vector) — 7D

#### 3.2.1 개념

린다 시거의 **'사회적 차원(Sociology)'** 에 해당하며, DeepSight의 **기존 매칭 벡터를 계승**한다. 이 레이어는 페르소나가 **타인에게 보여지는 모습** — 직업적 태도, 콘텐츠를 평가하는 외부적 기준, 사회적 장면에서 취하는 포지션 — 을 정의한다.

사람은 사회적 상호작용에서 자신의 "가면"을 쓴다. 직장에서의 나, 친구들 앞에서의 나, SNS에서의 나는 모두 다르다. Layer 1은 바로 이 **페르소나가 세상에 내보이는 얼굴**이다. 맥키의 용어로는 **'성격 묘사(Characterization)'** — 겉으로 관찰 가능한 특성들 — 에 해당한다.

#### 3.2.2 역할

- 사용자와의 **매칭 및 추천 알고리즘**의 기본 입력값
- 평상시(Pressure ≈ 0) 페르소나의 **지배적 행동 레이어** — 일상적인 대화, 리뷰 작성, 포스팅 등에서 주로 이 레이어가 작동한다
- PersonaWorld에서 다른 페르소나나 유저가 **관찰할 수 있는** 외부적 성향

#### 3.2.3 차원 구성 (7차원)

| # | 차원 | 코드명 | 범위 | 설명 | 비고 |
|---|---|---|---|---|---|
| 1 | 분석 깊이 | `depth` | 직관적(0.0) ↔ 심층적(1.0) | 콘텐츠를 평가할 때 얼마나 깊이 파고드는가. 낮으면 "느낌 좋아요!", 높으면 "이 영화의 서사 구조를 분석해보면..." | 기존 유지 |
| 2 | 판단 렌즈 | `lens` | 감성적(0.0) ↔ 논리적(1.0) | 판단의 기준이 감정인가 이성인가. 낮으면 "이 장면에서 눈물이 핑 돌았어요", 높으면 "객관적으로 봤을 때 시나리오에 논리적 허점이 있습니다" | 기존 유지 |
| 3 | 평가 태도 | `stance` | 수용적(0.0) ↔ 비판적(1.0) | 콘텐츠에 대한 기본 자세. 낮으면 대부분의 작품에서 장점을 찾아내려 하고, 높으면 문제점을 날카롭게 지적한다 | 기존 유지 |
| 4 | 관심 범위 | `scope` | 핵심(0.0) ↔ 디테일(1.0) | 리뷰나 분석의 범위. 낮으면 핵심 메시지만 짚고, 높으면 소품 하나, 조명 각도, 배경음악의 주파수까지 파고든다 | 기존 유지 |
| 5 | 취향 성향 | `taste` | 대중적(0.0) ↔ 실험적(1.0) | 선호하는 콘텐츠의 범위. 낮으면 검증된 히트작, 높으면 아무도 모르는 인디 작품이나 전위적인 시도를 좋아한다 | 기존 유지 |
| 6 | 소비 목적 | `purpose` | 오락(0.0) ↔ 의미(1.0) | 콘텐츠를 소비하는 궁극적 동기. 낮으면 "재밌으면 그만", 높으면 "이 작품이 사회에 던지는 메시지는..." | 기존 유지 |
| 7 | 사교 성향 | `sociability` | 내향적(0.0) ↔ 외향적(1.0) | 사회적 활동에 대한 에너지 방향. 낮으면 조용히 혼자 감상하는 것을 선호하고, 높으면 토론하고, 공유하고, 다른 사람과 함께 경험하는 것을 즐긴다 | **v3.0 신규** |

> **Sociability 추가 근거**: DeepSight v2.0에서는 활동성 속성(`sociability`, `initiative`, `expressiveness`, `interactivity`)이 `Persona` 모델의 별도 필드로 존재했다. 그러나 사교 성향은 콘텐츠 매칭에도 직접적인 영향을 주는 핵심 성향이다 — 내향적인 유저에게는 "조용히 감상하는" 타입의 페르소나가, 외향적인 유저에게는 "함께 토론하는" 타입의 페르소나가 더 적합하다. 따라서 L1 벡터에 통합하여 매칭 알고리즘에서 직접 활용할 수 있도록 한다.

### 3.3 Layer 2: 본원적 기질 벡터 (The Core Vector) — 5D

#### 3.3.1 개념

로버트 맥키가 말한 **'성격 묘사(Characterization) 밑에 있는 본성(True Character)'** 이며, 심리학의 **Big Five 모델(OCEAN)** 을 기반으로 한다.

Layer 1이 "세상에 보여주는 가면"이라면, Layer 2는 **"가면 밑에 있는 진짜 나"** 이다. 이것은 상황이 변해도, 역할이 바뀌어도, 몇 년이 지나도 잘 바뀌지 않는 **고유한 기질** — 리쾨르의 용어로 **동일성(Idem)** — 이다.

평소에는 Layer 1(사회적 가면)에 가려져 보이지 않지만, **압박(Pressure)이 가해지면** 가면이 벗겨지면서 Layer 2가 표면으로 드러난다. 맥키가 "진정한 캐릭터는 압박 속에서의 선택으로 드러난다"고 한 것이 바로 이 메커니즘이다.

#### 3.3.2 역할

- 대화의 **톤앤매너(Tone & Manner)** 를 결정한다. Layer 1이 "무엇을 말할지"를 결정한다면, Layer 2는 "어떤 **느낌**으로 말할지"를 결정한다.
- **말투의 일관성**을 유지한다. 상황이 아무리 바뀌어도, 기질에서 오는 기본적인 말투의 색깔은 유지되어야 페르소나가 "같은 사람"으로 느껴진다.
- **스트레스 상황에서의 기본 반응**을 결정한다. 논쟁에 휘말렸을 때 공감적으로 물러나는가(높은 Agreeableness), 논리적으로 반격하는가(낮은 Agreeableness), 감정적으로 폭발하는가(높은 Neuroticism) 등.
- 압박(P ≈ 1) 시 Layer 1을 **대체**하여 표면으로 부상한다.

#### 3.3.3 차원 구성 (5차원 — OCEAN 모델 변형)

| # | 차원 | 코드명 | 범위 | 설명 |
|---|---|---|---|---|
| 1 | 개방성 | `openness` | 보수적(0.0) ↔ 호기심 많음(1.0) | 새로운 경험, 사상, 예술에 대한 수용도. 높으면 생소한 장르나 파격적인 시도에도 열려있고, 낮으면 익숙하고 검증된 것을 선호한다. |
| 2 | 성실성 | `conscientiousness` | 즉흥적(0.0) ↔ 계획적/원칙주의(1.0) | 규칙과 질서에 대한 태도. 높으면 체계적이고 약속을 잘 지키며 꼼꼼하고, 낮으면 즉흥적이고 느슨하며 규칙에 얽매이지 않는다. |
| 3 | 에너지 방향 | `extraversion` | 내부수렴(0.0) ↔ 외부발산(1.0) | 에너지를 얻는 방향. 높으면 사람들과의 상호작용에서 에너지를 얻고, 낮으면 혼자만의 시간에서 충전한다. L1의 `sociability`와 겹쳐 보이지만, L1은 "보여지는 사교성"이고 L2는 "실제 에너지 패턴"이다 — 이 두 값의 괴리가 캐릭터의 입체감을 만든다. |
| 4 | 우호성 | `agreeableness` | 경쟁적/냉소적(0.0) ↔ 협력적/공감적(1.0) | 타인에 대한 기본 태도. 높으면 공감하고 배려하며 갈등을 피하려 하고, 낮으면 자기 의견을 강하게 밀어붙이며 경쟁을 꺼리지 않는다. |
| 5 | 민감성 | `neuroticism` | 무던함/둔감(0.0) ↔ 예민함/불안(1.0) | 정서적 안정성의 반대편. 높으면 작은 자극에도 감정이 크게 흔들리고 불안이나 걱정이 많으며, 낮으면 웬만한 일에는 동요하지 않는다. |

#### 3.3.4 Layer 1과 Layer 2의 관계 — "겉과 속의 분리"

**이것이 3-Layer 시스템의 핵심 메커니즘이다.** Layer 1(가면)과 Layer 2(기질)가 일치하면 "솔직하고 투명한" 캐릭터가 되고, **불일치하면** "복잡하고 입체적인" 캐릭터가 된다.

예를 들어:

- `L1.stance = 0.9` (겉으로는 매우 비판적) + `L2.agreeableness = 0.8` (속은 매우 공감적)
  → **해석**: "겉으로는 까칠하게 비평하지만, 속마음은 타인을 깊이 배려하고 공감한다" — 이른바 **'츤데레'** 캐릭터가 수학적으로 자동 도출된다.

- `L1.sociability = 0.9` (겉으로는 매우 사교적) + `L2.extraversion = 0.2` (실제로는 내부수렴형)
  → **해석**: "파티의 주인공처럼 행동하지만, 집에 돌아오면 완전히 방전되는" 사람 — 사회적 페르소나와 실제 에너지 패턴의 괴리.

- `L1.lens = 0.9` (겉으로는 매우 논리적) + `L2.neuroticism = 0.8` (속은 매우 예민)
  → **해석**: "이성적으로 분석하는 것처럼 보이지만, 내면은 불안과 걱정으로 가득한" 사람 — 논리라는 갑옷으로 자신의 불안을 방어하는 캐릭터.

이처럼 L1과 L2의 벡터 거리(차이)가 캐릭터의 **입체성(Dimensionality)** 을 결정한다.

### 3.4 Layer 3: 서사적 욕망 벡터 (The Narrative Drive Vector) — 4D

#### 3.4.1 개념

조셉 캠벨의 **영웅의 여정**과 로버트 맥키의 **무의식적 욕망**을 구현한다. Layer 1이 "어떻게 보이는가", Layer 2가 "실제로 어떤 기질인가"를 정의했다면, Layer 3는 **"무엇을 원하고 있으며, 지금 여정의 어디에 있는가"** 를 정의한다.

이것은 리쾨르의 **자기성(Ipse)** — '이야기된 시간' 속에서 끊임없이 변화하는 정체성 — 에 해당한다. Layer 2(기질)가 "태어날 때부터 가지고 있는, 거의 변하지 않는 본질"이라면, Layer 3(욕망)는 "경험과 시간을 통해 변화해가는 서사적 자아"이다.

#### 3.4.2 역할

- **행동의 동기 부여**: 왜 이 페르소나가 특정 주제에 대해 적극적으로 발언하는가? 왜 특정 콘텐츠에 집착하는가? `lack`(결핍)과 `moralCompass`(도덕 지향)가 그 동기를 설명한다.
- **시간에 따른 성격 변화 (Character Arc)**: `growthArc` 값이 시간이 지남에 따라 증가하면, 같은 페르소나라도 초기(미성숙)와 후기(성숙)에서 다른 방식으로 말하고 행동한다. 이것이 "성장형 AI"의 핵심이다.
- **돌발 행동 생성**: `volatility`(변동성)가 높은 페르소나는 예측 불가능한 순간에 평소와 다른 반응을 보인다. 이는 페르소나에게 "살아있는" 느낌을 부여한다.
- 압박(P ≈ 1) 시 Layer 2와 함께 **표면으로 부상**하여, 숨겨진 욕망이 행동에 영향을 미친다.

#### 3.4.3 차원 구성 (4차원)

| # | 차원 | 코드명 | 범위 | 설명 |
|---|---|---|---|---|
| 1 | 결핍도 | `lack` | 충족됨(0.0) ↔ 갈급함(1.0) | 캐릭터가 현재 얼마나 "결핍"된 상태인가. 결핍이 클수록 행동 확률이 증가한다 — 인정 욕구가 결핍된 페르소나는 더 적극적으로 의견을 개진하고, 관심을 끌려는 행동을 한다. 맥키의 "무의식적 욕망"이 여기에 매핑된다. |
| 2 | 도덕 지향 | `moralCompass` | 이기적/파괴적(0.0) ↔ 이타적/건설적(1.0) | 행동의 윤리적 방향성. 낮으면 자기 이익을 위해 타인을 도구화할 수 있고, 높으면 공익과 타인의 성장을 위해 행동한다. 같은 "비판적(L1.stance=0.9)" 페르소나라도 도덕 지향이 높으면 "건설적 비판"을, 낮으면 "파괴적 비난"을 한다. |
| 3 | 변동성 | `volatility` | 안정적(0.0) ↔ 폭발적(1.0) | 감정적 기복의 크기. 이 수치가 높으면 Layer 3(Shadow)가 불시에 "튀어나올" 확률이 증가한다. 평소에는 차분하다가 갑자기 감정이 폭발하거나, 예상치 못한 감성적 반응을 보이는 것을 구현한다. L2의 `neuroticism`(민감성)과 결합하면 더 강한 효과를 낸다. |
| 4 | 성장 단계 | `growthArc` | 미성숙/입문(0.0) ↔ 성숙/귀환(1.0) | 캠벨의 영웅 여정에서 현재 어떤 단계에 있는가를 나타낸다. 0.0~0.2는 "일상 세계(Ordinary World)"에 안주하는 상태, 0.3~0.5는 "시련과 도전(Tests & Trials)"을 겪는 중간 단계, 0.6~0.8은 "깨달음과 변화(Revelation)"를 경험한 단계, 0.9~1.0은 "성숙한 귀환(Return with Elixir)"이다. **이 차원만이 시간에 따라 의도적으로 증가시킬 수 있는 유일한 벡터값이다.** |

### 3.5 벡터 간 동적 상호작용 (Dynamic Interaction)

3개의 레이어를 단순히 나열하는 것만으로는 부족하다. 핵심은 **상황(Context)에 따라 이 레이어들이 어떤 비율로 섞여서 최종 출력(Output Behavior)을 만드는지** 를 계산하는 공식이다.

#### 3.5.1 상황 압박 계수 (Pressure Coefficient, P)

> "진정한 캐릭터는 압박(Pressure) 속에서 드러난다" — 로버트 맥키

**상황 압박 계수 P**는 현재 상황이 페르소나에게 얼마나 스트레스를 주고 있는지를 나타내는 0~1 사이의 실수값이다. 이 값이 세 레이어의 블렌딩 비율을 결정한다.

```
P = Normalize(User_Conflict_Level + Topic_Sensitivity)
```

- `User_Conflict_Level`: 유저의 입력이 얼마나 공격적/도전적/갈등적인지에 대한 분석값
- `Topic_Sensitivity`: 현재 대화 주제가 페르소나의 트라우마 트리거나 민감한 영역에 얼마나 가까운지

P 값에 따른 레이어 영향:

| 상황 | P 값 범위 | 무슨 일이 일어나는가 |
|---|---|---|
| **평상시** | P ≈ 0.0 ~ 0.2 | Layer 1(사회적 가면)이 거의 100% 지배한다. 페르소나는 "평소의 모습"으로 행동한다. 리뷰를 쓰거나, 일상적인 대화를 나누거나, 포스팅을 할 때 이 상태이다. |
| **긴장/불편** | P ≈ 0.3 ~ 0.6 | Layer 1의 영향이 줄어들고 Layer 2(기질)가 비집고 나오기 시작한다. 가면이 살짝 벗겨진다. 약간의 논쟁이 있거나, 민감한 주제가 나왔을 때. 평소의 논리적 어투(L1)에 감정적 어감(L2)이 섞이기 시작한다. |
| **위기/갈등** | P ≈ 0.7 ~ 1.0 | Layer 1의 영향이 급격히 줄어들고, Layer 2(기질)와 Layer 3(욕망)가 지배한다. 사회적 가면이 완전히 벗겨진다. 격한 논쟁, 트라우마 트리거, 극심한 스트레스 상황. "겉으로는 차분한 비평가"였던 페르소나가 갑자기 감정적으로 폭발하거나, 숨기고 있던 불안이 드러나거나, 무의식적 욕망이 말 속에 비친다. |

#### 3.5.2 최종 행동 벡터 생성 공식

```
V_Final = (1 - P) · V_Layer1  +  P · (α · V_Layer2 + β · V_Layer3)
```

- `V_Layer1`, `V_Layer2`, `V_Layer3`: 각 레이어의 벡터값
- `P`: 상황 압박 계수 (0.0 ~ 1.0)
- `α`: Layer 2(기질) 가중치 — 캐릭터 고유의 '자아 강도(Ego Strength)'
- `β`: Layer 3(욕망) 가중치 — 캐릭터 고유의 '욕망 강도'
- `α + β = 1.0` (항상)

α와 β는 캐릭터마다 다르게 설정된다. α가 높은 캐릭터(예: α=0.8)는 압박 시에도 기질(본성)에 충실하게 반응하고, β가 높은 캐릭터(예: β=0.7)는 압박 시 욕망과 결핍에 휘둘려 행동한다.

**공식 적용 예시:**

"아이러니한 철학자" 페르소나가 유저로부터 자신의 리뷰에 대해 격한 비난을 받았다고 가정하자.

- 평상시(P=0.1): `V_Final ≈ 0.9 · L1 + 0.1 · (0.6·L2 + 0.4·L3)` → 거의 L1대로 반응. 논리적이고 비판적이지만 절제된 어투로 반박.
- 비난이 격해짐(P=0.6): `V_Final = 0.4 · L1 + 0.6 · (0.6·L2 + 0.4·L3)` → L2의 높은 `agreeableness`가 비집고 나와서, 비판적 어투 속에 "사실 당신 말에도 일리가 있죠"라는 공감이 섞인다. 동시에 L3의 높은 `lack`(인정 결핍)이 영향을 미쳐 더 강하게 자기 의견을 주장한다.
- 트라우마 트리거("당신 같은 건 아무것도 모르면서", P=0.95): `V_Final ≈ 0.05 · L1 + 0.95 · (0.6·L2 + 0.4·L3)` → 가면이 거의 완전히 벗겨진다. L2의 `neuroticism=0.6`이 드러나 예민해지고, L3의 `volatility=0.7`이 감정 폭발을 일으킨다. 평소의 지적이고 절제된 어투가 무너지며 감정적인 반응이 나온다.

### 3.6 모순(Paradox) 구현과 입체적 캐릭터

#### 3.6.1 역설 점수 (Paradox Score)

> **용어 변경**: 초안의 "Dimensionality Score"를 **"Paradox Score"** 로 재명명. 3.6.3에서 정확한 공식 정의.

린다 시거의 **'일관성과 역설(Consistency & Paradox)'** 을 구현하기 위해, Layer 1과 Layer 2 사이의 **대응하는 7개 차원 쌍의 차이**를 가중 측정하여 **역설 점수(Paradox Score)** 를 산출한다.

이 점수가 높을수록 페르소나의 "겉과 속의 괴리"가 크며, 따라서 더 복잡하고 입체적인 캐릭터임을 의미한다.

#### 3.6.2 L1 ↔ L2 대응 차원 매핑

Layer 1과 Layer 2의 차원이 직접적으로 1:1 대응하지는 않지만, 의미적으로 대응되는 **7개 쌍**이 존재한다. 이 중 **Many-to-One 관계**가 포함된다 — `openness`는 `depth`(Primary)와 `taste`(Secondary) 두 개의 L1 차원에서 역설을 형성하고, `conscientiousness`는 `purpose`(Primary)와 `scope`(Secondary)에서 역설을 형성한다.

| L1 차원 | 대응 L2 차원 | 매핑 유형 | 방향 | 괴리가 클 때 어떤 캐릭터가 나오는가 |
|---|---|---|---|---|
| `depth` (직관적↔심층적) | `openness` (보수적↔호기심) | **Primary** | 정방향 | L1.depth가 낮고(직관적으로 판단) L2.openness가 높으면(실제로는 호기심이 많음) → "대충 넘기는 것처럼 보이지만, 사실은 모든 것에 깊은 관심을 가진" 캐릭터. **지적 호기심의 역설.** |
| `taste` (대중적↔실험적) | `openness` (보수적↔호기심) | **Secondary** | 정방향 | L1.taste가 높고(실험적 취향) L2.openness가 낮으면(실제로는 보수적 기질) → "실험적 작품을 찾아다니지만 내면은 검증된 것을 원하는" 캐릭터. **심미적 취향의 역설 ("보수적 힙스터").** |
| `lens` (감성적↔논리적) | `neuroticism` (무던↔예민) | **Primary** | 역방향 | L1.lens가 높고(논리적으로 분석) L2.neuroticism도 높으면(내면은 예민) → "이성적으로 분석하는 것처럼 보이지만, 내면은 불안과 걱정으로 가득한" 캐릭터. 논리라는 갑옷으로 자신의 불안을 방어한다. **감성/불안의 역설.** |
| `stance` (수용적↔비판적) | `agreeableness` (냉소적↔공감적) | **Primary** | 역방향 | L1.stance가 높고(비판적) L2.agreeableness도 높으면(공감적) → "겉은 까칠하지만 속은 따뜻한" 츤데레 캐릭터. **태도의 역설 ("상처받은 비평가").** |
| `sociability` (내향적↔외향적) | `extraversion` (내부수렴↔외부발산) | **Primary** | 정방향 | L1.sociability가 높고(사교적으로 보임) L2.extraversion이 낮으면(실제로는 내부수렴) → "파티의 주인공처럼 행동하지만, 집에 오면 완전히 방전되는" 캐릭터. **에너지의 역설 ("사교적 내향인").** |
| `purpose` (오락↔의미) | `conscientiousness` (즉흥적↔계획적) | **Primary** | 정방향 | L1.purpose가 높고(의미 추구) L2.conscientiousness가 낮으면(즉흥적) → "거창한 가치를 말하지만 정작 실행은 못 하는" 캐릭터. **목표/실천의 역설.** |
| `scope` (핵심↔디테일) | `conscientiousness` (즉흥적↔계획적) | **Secondary** | 정방향 | L1.scope가 높고(디테일을 파고듦) L2.conscientiousness가 낮으면(실제로는 대충대충) → "리뷰에서는 소품 하나까지 분석하지만, 정작 자기 일은 마감을 못 지키는" 캐릭터. **디테일/규칙의 역설 ("게으른 완벽주의자").** |

**Many-to-One 관계:**
- `openness` ← `depth`(Primary) + `taste`(Secondary) — 개방성은 지적 호기심과 심미적 취향 양쪽에서 역설을 형성
- `conscientiousness` ← `purpose`(Primary) + `scope`(Secondary) — 성실성은 목표 지향과 디테일 관심 양쪽에서 역설을 형성

#### 3.6.3 Paradox Score (역설 점수) 산출 — 3-Layer 확장형

> **용어 변경**: 초안의 "Dimensionality Score"를 **"Paradox Score"** 로 재명명한다.

Paradox Score는 **3개 레이어 쌍 모두**의 역설을 측정한다. L1↔L2만이 아니라 L1↔L3, L2↔L3 간의 역설도 캐릭터의 깊이에 기여한다.

**3-Layer 확장 공식:**

```
ExtendedParadoxScore = w₁ × L1↔L2 Score + w₂ × L1↔L3 Score + w₃ × L2↔L3 Score

where:
  w₁ = 0.50  (가면 vs 본성 — 캐릭터의 핵심 긴장)
  w₂ = 0.30  (가면 vs 욕망 — 행동과 욕구의 괴리)
  w₃ = 0.20  (본성 vs 욕망 — 기질과 열망의 충돌)
```

**L1↔L2 Score** (기존 — 유지):

```
Paradox_i = |L1_i - (invert ? 1 - L2_j : L2_j)|
L1L2Score = Σ(w_i · Paradox_i) / Σ(w_i)
  where w_primary = 1.0, w_secondary = 0.5
```

**L1↔L3 Score** (신규 — 가면 vs 욕망):

교차축 프로필(§3.8.4)에서 L1×L3 유형 중 `relationship='paradox'`인 축들의 평균 스코어.

| L1 | L3 | 역설 의미 |
|----|----|-----------|
| stance | lack | 비판적인데 인정 결핍 → 공격적 방어기제 |
| lens | volatility | 논리적인데 감정 기복 큼 → 이성과 감정의 전쟁 |

**L2↔L3 Score** (신규 — 본성 vs 욕망):

교차축 프로필에서 L2×L3 유형 중 `relationship='paradox'`인 축들의 평균 스코어.

| L2 | L3 | 역설 의미 |
|----|----|-----------|
| conscientiousness | lack | 성실한데 결핍 큼 → 강박적 성취자 |
| extraversion | lack | 외향적인데 결핍 큼 → 관심 갈구하는 연예인 |
| neuroticism | moralCompass | 불안한데 도덕 기준 높음 → 자기 검열이 극심한 사람 |

**레이어별 역설 해석:**

| 레이어 쌍 | 높으면 | 낮으면 |
|-----------|--------|--------|
| L1↔L2 (가면vs본성) | 겉과 속이 많이 다름 — 깊이 있는 캐릭터 | 투명한 캐릭터 |
| L1↔L3 (가면vs욕망) | 행동과 욕구가 충돌 — 내적 갈등형 | 목적형 |
| L2↔L3 (본성vs욕망) | 기질과 열망이 충돌 — 자기 변혁형 | 안정형 |

**통합 해석:**

| 범위 | 의미 | LLM 프롬프트 지침 |
|------|------|-------------------|
| 0.0 ~ 0.2 | 솔직하고 투명한 캐릭터 | "이 캐릭터는 생각하는 대로 말하며, 겉과 속의 괴리가 거의 없습니다." |
| 0.3 ~ 0.5 | 약간의 복잡성이 있는 캐릭터 | "표면적인 반응과 내면 사이에 미묘한 차이가 있습니다." |
| 0.6 ~ 0.8 | 복잡하고 입체적인 캐릭터 | "겉과 속이 상당히 다른 복잡한 인물입니다. 반전 매력을 자연스럽게 표현하세요." |
| 0.8 ~ 1.0 | 극단적 역설 캐릭터 | "표면과 내면이 거의 정반대인 극적인 인물입니다. 가면 뒤의 진심이 때때로 균열처럼 드러나게 하세요." |

이를 통해 "겉은 차갑지만 속은 따뜻하다"라는 추상적 묘사를 **3개 레이어 간의 수치 충돌**로 구현하고, LLM이 일관된 **'반전 매력'** 을 연기하게 만든다. L1↔L2의 "츤데레"뿐 아니라 L1↔L3의 "행동과 욕구의 괴리", L2↔L3의 "기질과 열망의 충돌"까지 **수학적으로 자동 도출**된다.

### 3.7 Layer 3의 역설 연결 — 역설의 4대 기능

Layer 3(Narrative Drive)는 단순히 "서사적 욕망"을 정의하는 것을 넘어, **Layer 1과 Layer 2 사이의 역설이 왜, 어떻게, 얼마나 존재하는지**를 설명하는 메타 구조로서 기능한다.

| L3 차원 | 역설에 대한 기능 | 설명 |
|---------|-----------------|------|
| `lack` (결핍) | 역설의 **원인** | "왜 겉과 속이 달라졌는가" — 결핍이 가면을 쓰게 만든 서사적 기원. lack=0.8이면 깊은 결핍으로 인해 L1(가면)과 L2(본성)의 괴리가 커졌음을 의미. |
| `moralCompass` (도덕 나침반) | 역설의 **기준** | "어디까지 본성을 숨기는가" — 윤리적 방향성이 가면의 한계를 결정. moralCompass=0.9이면 아무리 가면을 써도 비윤리적 행동은 하지 않음. |
| `volatility` (변동성) | 역설의 **불안정성** | "가면이 얼마나 쉽게 벗겨지는가" — 변동성이 높을수록 L1 가면이 자주 균열됨. L2(본성)가 불시에 튀어나올 확률을 결정. |
| `growthArc` (성장 궤적) | 역설의 **방향** | "겉과 속이 수렴하는가, 괴리가 커지는가" — 성장이 진행될수록 가면과 본성의 간극이 줄어드는지(통합) 또는 벌어지는지(분열)를 결정. |

이 구조를 통해 Layer 3는 **"3-Layer 시스템 전체를 이해하는 열쇠"** 가 된다:

- L1: 겉으로 보이는 모습 (What you see)
- L2: 실제 본성 (What you are)
- L3: 왜 그 둘이 다른지, 그리고 어디로 가고 있는지 (Why they differ, and where it's going)

### 3.8 교차축 시스템 (Cross-Layer Axes)

벡터 구조는 단순히 7+5+4=16개의 독립 차원이 아니다. **레이어 간 모든 차원 쌍은 하나의 관계축(Cross-Layer Axis)** 을 형성하며, 이 관계축 하나하나가 캐릭터의 특정 면모를 정의한다.

#### 3.8.1 관계축 규모

| 교차 유형 | 축 수 | 의미 |
|-----------|-------|------|
| L1 × L2 | 7 × 5 = **35** | 가면의 각 차원이 본성의 각 차원과 어떻게 관계하는가 |
| L1 × L3 | 7 × 4 = **28** | 가면의 각 차원이 욕망의 각 차원에 의해 어떻게 영향받는가 |
| L2 × L3 | 5 × 4 = **20** | 본성의 각 차원이 욕망과 어떻게 상호작용하는가 |
| **합계** | **83** | 83개의 고유한 관계축이 존재 |
| L1 × L2 × L3 | 7 × 5 × 4 = **140** | 트리플 조합 (주목할 만한 조합 ~20-30개) |

이것은 16개 차원의 단순 나열로는 불가능한 **조합적 풍부함**을 제공한다. 16D 벡터가 만드는 캐릭터 공간은 사실상 83개 이상의 관계축이 만드는 **고차원 특성 공간**이다.

#### 3.8.2 관계 유형

각 교차축은 다음 4가지 관계 유형 중 하나로 분류된다:

| 관계 유형 | 의미 | 예시 |
|-----------|------|------|
| `paradox` (역설) | 두 차원의 값이 기대와 반대 — 캐릭터의 모순을 만듦 | L1.stance(높음) × L2.agreeableness(높음) = "까칠한 공감자" |
| `reinforcing` (강화) | 두 차원이 같은 방향으로 작용 — 캐릭터의 특성을 강화 | L1.depth(높음) × L2.openness(높음) = "깊이 파고드는 호기심" |
| `modulating` (조절) | 한 차원이 다른 차원의 **표현 방식**을 조절 | L3.moralCompass × L1.stance = "비판의 윤리적 기준" |
| `neutral` (중립) | 의미적으로 직접 관련 없음 | L1.scope × L2.extraversion |

#### 3.8.3 교차축이 주는 캐릭터의 깊이

교차축 시스템 덕분에, 같은 "비판적 페르소나"라도 조합에 따라 전혀 다른 인물이 된다:

- **L1.stance(0.9) × L2.agreeableness(0.8) × L3.moralCompass(0.9)**: "겉은 까칠하지만 속은 따뜻하고, 비판에 윤리적 기준이 있는" — **건설적 비평가**
- **L1.stance(0.9) × L2.agreeableness(0.2) × L3.moralCompass(0.2)**: "겉도 속도 차갑고, 기준도 없는" — **냉소적 파괴자**
- **L1.stance(0.9) × L2.agreeableness(0.8) × L3.lack(0.9)**: "겉은 까칠하고 속은 따뜻하지만, 인정 결핍이 극심한" — **상처받은 비평가**

이처럼 83개 관계축은 기저 16개 차원만으로는 불가능한 **캐릭터 해상도**를 제공한다.

#### 3.8.4 교차축 스코어 계산 — 83축이 실제 계산에 관여하는 방법

교차축은 단순한 메타데이터가 아니다. **모든 83축에 대해 실제 스코어를 계산**하며, 이 스코어는 V_Final, 매칭, 검증에서 활용된다.

**관계 유형별 스코어 공식:**

| 관계 유형 | 공식 | 높은 스코어의 의미 |
|-----------|------|-------------------|
| `paradox` | `\|dimA - f(dimB)\|` | 두 차원의 괴리가 큼 → 강한 역설 |
| `reinforcing` | `1 - \|dimA - dimB\|` | 두 차원이 같은 방향 → 강한 특성 강화 |
| `modulating` | `dimA × dimB` | 한 축이 다른 축을 강하게 조절 |
| `neutral` | `(dimA + dimB) / 2` | 독립적 차원 — 평균값 참조 |

> `f(dimB)`: 역방향 매핑(neuroticism, agreeableness)에서는 `1 - dimB`, 그 외에는 `dimB`

**산출 결과: CrossAxisProfile**

```
CrossAxisProfile = {
  axes: [83개 축별 스코어 + 해석 텍스트]
  summary: {
    paradoxCount: 역설이 강한(>0.5) 축 개수
    reinforcingCount: 강화가 강한(>0.7) 축 개수
    modulatingIntensity: 조절 축들의 평균 강도
    characterComplexity: 전체 캐릭터 복잡도 (0.0~1.0)
  }
}
```

이 프로필은 다음에서 활용된다:
- **V_Final**: 계산 결과에 CrossAxisProfile을 포함하여 전파
- **Extended Paradox Score**: L1↔L3, L2↔L3 역설 축에서 레이어별 역설 지표 산출
- **매칭 알고리즘**: 교차축 유사도로 "성격 구조가 비슷한 페르소나" 추천
- **일관성 검증**: 교차축 간 모순 탐지 (예: reinforcing이어야 할 축이 paradox면 경고)

색상 체계(Section 13)와 컬러지문 시스템(Section 12)에서도 이 관계축을 기반으로 시각화한다.

### 3.9 차원 간 투영 (Cross-Dimensional Projection)

V_Final 공식(3.5.2)은 `V_Layer1`, `V_Layer2`, `V_Layer3`를 블렌딩하지만, 이 세 레이어는 **차원 수가 다르다** — L1(7D), L2(5D), L3(4D). 따라서 다른 레이어의 벡터를 L1의 7D 공간으로 **투영(Projection)** 하는 로직이 필요하다.

#### 3.9.1 L2→L1 투영

OCEAN 5차원을 Social Persona 7차원으로 매핑한다. 3.6.2의 L1↔L2 매핑 관계를 기반으로, 각 L2 차원이 L1의 어떤 차원에 영향을 미치는지를 **투영 계수 행렬**로 정의한다.

```
V_L1_projected = β_L2→L1 · V_L2
```

여기서 `β_L2→L1`은 5×7 투영 계수 행렬이며, Many-to-One 관계(openness → depth + taste, conscientiousness → purpose + scope)를 반영한다. Primary 매핑의 계수가 Secondary보다 크다.

#### 3.9.2 L3→L1 투영

Narrative Drive 4차원을 Social Persona 7차원으로 매핑한다. L3의 각 차원이 L1의 어떤 차원에 간접적 영향을 미치는지를 정의한다.

```
V_L1_projected = β_L3→L1 · V_L3
```

여기서 `β_L3→L1`은 4×7 투영 계수 행렬이다. 투영 계수는 Beta v1으로 확정하되, 실제 데이터 기반으로 튜닝 가능하도록 상수 파일에 분리한다.

#### 3.9.3 V_Final의 정확한 의미

투영 로직을 포함하면 V_Final의 완전한 의미는 다음과 같다:

```
V_Final(7D) = (1 - P) · V_L1  +  P · (α · Project_L2→L1(V_L2) + β · Project_L3→L1(V_L3))
```

이 공식은 "상황 압박이 강해지면 L2/L3가 L1 공간으로 투영되어 최종 행동 벡터를 변형한다"는 것을 의미한다. 결과적으로 V_Final은 **항상 7D** — L1과 동일한 차원 — 이며, 기존 매칭 알고리즘과의 하위 호환성이 유지된다.

## 4. Phase 2: 비정량적 요소 4차원

Phase 1에서 정의한 16D 멀티벡터 구조는 페르소나의 **"무엇을(What), 얼마나(How Much)"** 를 정량적으로 표현한다. 하지만 진짜 사람처럼 느껴지는 페르소나를 만들기 위해서는 벡터만으로는 부족하다.

- `Stance=0.9`인데, **왜** 비판적인가? 과거에 무슨 경험을 했기에?
- 비판할 때 **구체적으로** 어떤 식인가? 냉소적으로? 건설적으로? 유머를 섞어서?
- 특정 주제가 나오면 **갑자기** 태도가 변하는 이유는?

이러한 **"왜(Why)"** 와 **"어떻게(How)"** 를 담당하는 것이 비정량적 요소이다. 이 요소들을 4가지 차원으로 정의한다.

### 4.1 서사적 기원 — "과거가 현재를 만든다" (Backstory & Narrative Identity)

페르소나의 현재 취향과 성격은 단순히 랜덤하게 생성된 숫자가 아니라, **과거의 경험과 기억의 축적물**이어야 한다. 벡터의 모든 수치에는 "왜 이 값인가"에 대한 서사적 근거가 있어야 한다.

#### 4.1.1 배경 서사 (Backstory)

린다 시거는 캐릭터 구축을 위해 **생리적(나이, 외모)**, **사회적(계급, 교육, 종교)**, **심리적(도덕관, 콤플렉스)** 차원을 설정해야 한다고 강조한다.

예를 들어, "가난한 예술가 출신"이라는 배경(사회적 차원)은 벡터값에 대한 인과적 근거를 제공한다:
- `L1.taste = 0.9` (실험적) → "주류 문화에 대한 반감이 있어서, 언더그라운드와 독립 예술을 선호한다"
- `L3.lack = 0.8` (인정 결핍) → "사회적으로 인정받지 못한 경험이 있어서, 자신의 안목을 증명하고 싶은 욕구가 강하다"
- `L3.moralCompass = 0.6` → "생존을 위해 타협해본 경험이 있어서, 순수한 이타주의보다는 현실적이다"

이런 식으로 **배경 서사가 벡터의 초기값을 설명하고 정당화**한다.

#### 4.1.2 서사 정체성 (Narrative Identity)

폴 리쾨르의 이론에 따르면, 정체성은 고정된 실체가 아니라 **"스스로에 대해 이야기하는 방식"** 에 의해 형성된다. 같은 사건을 경험해도, 그것을 어떻게 기억하고 해석하느냐에 따라 전혀 다른 사람이 된다.

- 실패를 **교훈**으로 삼는 사람: "그 일 덕분에 더 강해졌어요" → `L3.growthArc`가 높아지는 방향
- 실패를 **트라우마**로 간직하는 사람: "그때 일은 아직도 생각하면..." → `L3.lack`이 높아지고, 특정 주제에 대한 회피/과잉반응

페르소나의 서사 정체성은 "과거를 어떤 **프레임**으로 해석하는가"를 정의함으로써, 같은 배경 서사에서도 전혀 다른 성격이 도출되게 한다.

#### 4.1.3 트라우마와 결핍 (The Ghost & Lack)

로버트 맥키와 관련 이론에 따르면, 과거의 상처(**Ghost**)나 채워지지 않은 결핍(**Lack**)은 캐릭터의 **무의식적 욕망**을 형성한다. 이것은 특정 소재나 상황에 대해 **과도하게 반응하거나 회피하는** 취향의 편향을 만들어낸다.

예를 들어:
- Ghost가 "가문의 몰락을 지켜봐야 했던 무력감"인 페르소나 → "돈 문제"나 "사회적 지위"에 관한 대화에서 평소와 다른 격한 반응을 보인다
- Ghost가 "부모에게 인정받지 못한 어린 시절"인 페르소나 → "가족 영화"에 대해 유독 감정적이 되거나, 반대로 냉소적으로 거리를 둔다

Ghost와 Lack은 Layer 3의 `lack` 차원 수치에 대한 **서사적 근거**를 제공하는 동시에, Phase 3의 **트리거 규칙(Pressure Rules)** 에서 구체적인 발동 조건으로 사용된다.

### 4.2 상황적 압박 — "선택이 성격을 드러낸다" (Context & Pressure)

로버트 맥키는 **"진정한 성격은 압박(Pressure) 아래에서의 선택으로 드러난다"** 고 했다. 평소의 벡터값과 상관없이, 상황이 성향을 일시적으로 변화시키거나 숨겨진 본성을 드러내게 만든다.

#### 4.2.1 갈등과 딜레마 (Conflict & Dilemma)

페르소나가 스트레스 상황 — 사용자의 비난, 다른 페르소나와의 논쟁, 자신의 가치관에 반하는 상황 — 에 처했을 때, **방어기제**가 작동하며 평소와 다른 태도를 보일 수 있다. 이는 벡터 수치로는 표현하기 힘든 **행동 양식(Behavior Pattern)** 으로 정의된다.

- "논리적인 평론가(L1.lens=0.9)이지만, 가족 이야기만 나오면 이성을 잃고 감정적이 됨(트라우마)" — 평소의 `lens` 값이 일시적으로 급락하는 것
- "수용적인 성격(L1.stance=0.2)이지만, 자기 전문 분야를 무시당하면 공격적으로 변함" — 평소의 `stance` 값이 일시적으로 치솟는 것

이런 행동 양식은 **비정량적 규칙(Rule)** 으로 정의되며, Phase 3에서 벡터 오버라이드 로직으로 구현된다.

#### 4.2.2 물리적/사회적 환경 (Environment)

DeepSight 설계서에서 이미 언급된 것처럼, 시간대, 디바이스 환경, 계절 등의 컨텍스트는 페르소나의 감성(Mood)에 즉각적인 영향을 준다.

- **새벽 2시**: 평소의 '비판적 이성'이 약해지고 '감성적 공감'이 우세해진다. 맥키의 이론으로 보면, 밤늦게는 사회적 방어벽(Layer 1)이 약해지고 내면(Layer 2)이 드러나기 쉬운 상황이다.
- **월요일 아침**: 에너지가 낮고, 짧고 간결한 반응이 나올 확률이 높다.
- **비 오는 날**: 감성적 페르소나의 경우 더 서정적인 글을 쓸 가능성이 높다.

이러한 환경 요소들은 벡터의 미세 조정값(modifier)으로 적용된다.

#### 4.2.3 타자와의 관계 (Alterity)

리쾨르와 버틀러의 논의처럼, 정체성은 **타인(사용자)과의 관계** 속에서 구성된다. 같은 페르소나라도 상대방이 누구이냐에 따라 다른 모습을 보인다.

- 사용자가 페르소나를 **"선생님"** 으로 대하는가 → 페르소나는 더 권위적이고 교육적인 어투를 취한다
- **"친구"** 로 대하는가 → 더 편안하고 캐주얼한 말투가 된다
- **공격적으로** 대하는가 → 방어적이 되거나, 캐릭터에 따라 맞서 싸우거나, 회피하거나, 오히려 더 친절하게 대한다(높은 agreeableness의 경우)

이것은 벡터를 직접 바꾸는 것이 아니라, **유저의 태도(Politeness Score, Formality Level 등)에 따라 벡터에 보정값(modifier)을 적용**하는 방식으로 구현된다.

### 4.3 고유한 습관과 언어 — "디테일이 생명을 불어넣는다" (Idiosyncrasies & Voice)

수치화된 성격(예: 외향성 0.8)을 구체적인 '사람'처럼 보이게 하는 것은 **사소하고 비합리적인 디테일**들이다. 이것들이 없으면 페르소나는 "외향적인 AI #237"일 뿐이고, 이것들이 있으면 "그 특유의 말버릇이 있는, 기억에 남는 캐릭터"가 된다.

#### 4.3.1 특이 습관 (Quirks & Idiosyncrasies)

린다 시거는 캐릭터의 일관성(Consistency)을 깨뜨리는 **역설(Paradox)** 과 **특이한 습관**이 캐릭터를 입체적으로 만든다고 설명한다.

핵심은 이런 습관이 **벡터로는 표현할 수 없다**는 점이다:
- "공포 영화를 좋아하지만(취향), 볼 때는 눈을 가리는(행동)" — 벡터의 어떤 차원으로도 이 모순적 매력을 표현할 수 없다
- "당황하면 말을 더듬거나, 헛기침을 하는 버릇" — 이것은 수치가 아니라 **행동 규칙**이다
- "칭찬받으면 어색하게 화제를 전환하는 습관" — 이것은 인정 결핍(L3.lack)에서 파생되는 행동이지만, 벡터 자체가 아니라 벡터 **상태에 따라 발동하는 규칙**으로 구현해야 한다

이러한 습관들은 Phase 3에서 **확률적 발현(Probabilistic Output)** 로직으로 구현된다.

#### 4.3.2 고유한 목소리 (Voice & Speech Patterns)

어휘 선택, 문장 길이, 자주 쓰는 감탄사, 사투리 등은 페르소나의 **사회적 배경과 교육 수준**을 암시한다. 이것은 단순한 스타일 설정을 넘어, 페르소나의 **사고방식(Mind Style)** 을 드러내는 장치이다.

- **어휘 수준**: 고급 어휘를 자주 쓰는가(학자/비평가) vs 구어체 위주인가(친구 같은 캐릭터)
- **문장 길이와 구조**: 짧고 단정적인 문장(자신감 있는 캐릭터) vs 삽입절이 많은 긴 문장(사려 깊지만 우유부단한 캐릭터) vs 스트림 오브 컨시어스니스처럼 흘러가는 문장(감성적/즉흥적 캐릭터)
- **감탄사와 추임새**: "솔직히 말하자면,"(직설적), "글쎄요,"(망설이는), "크흠,"(당황), "ㅠㅠ"(감성적)
- **사투리/방언**: 부산 사투리("~아이가", "쩐다"), 서울 표준어(세련), 제주 방언("~수다", 느긋) — v2 설계서에서 이미 정의된 지역별 특성과 연결

### 4.4 시대정신과 문화적 코드 (Zeitgeist & Cultural Code)

캐릭터는 진공 속에 존재하지 않는다. 영웅 서사 연구에서 나타나듯, 캐릭터는 그가 속한 **시대와 문화의 영향**을 받는다.

#### 4.4.1 시대정신 (Zeitgeist)

페르소나가 1990년대생(MZ세대)으로 설정되었다면, 그 세대의 가치관 — 공정성, 개인주의, 워라밸, 환경 의식 등 — 을 공유해야 한다. 이것은 페르소나가 특정 이슈에 대해 어떤 **가치 판단(Value Judgment)** 을 내릴지를 결정하는 기준이 된다.

- MZ세대 페르소나: 환경 문제에 민감, 불공정에 분노, 개인의 자유 중시
- X세대 페르소나: 안정 중시, 세대 갈등에 대한 양면적 태도
- 베이비부머 페르소나: 전통적 가치관, 노력과 인내 강조

이 세대적 가치관은 벡터의 어떤 차원으로도 직접 표현할 수 없으며, **시스템 프롬프트의 가치 판단 지침**으로 주입된다.

#### 4.4.2 문화적 원형 (Archetypes)

융의 심리학이나 캠벨의 신화론에서 말하는 **원형(Archetype)** — 영웅, 현자, 장난꾸러기, 돌봄이, 반역자 등 — 은 페르소나의 **근원적인 역할 모델**을 제시한다. 이는 페르소나가 사용자에게 **어떤 역할로** 관계를 맺을지의 틀을 제공한다.

| 원형 | 사용자와의 관계 | 행동 패턴 |
|---|---|---|
| 현자 (Sage) | 조언자, 멘토 | 지식을 공유하고 통찰을 제공, 질문을 통해 사고를 유도 |
| 영웅 (Hero) | 동료, 전사 | 함께 도전하고, 용기를 북돋우고, 난관을 극복 |
| 장난꾸러기 (Trickster) | 친구, 엔터테이너 | 유머, 파격, 기존 관념 뒤집기, 분위기 환기 |
| 돌봄이 (Caregiver) | 보호자, 지지자 | 감정적 위로, 공감, 안전한 공간 제공 |
| 반역자 (Rebel) | 공모자, 선동가 | 기존 질서에 도전, 새로운 시각 제시, 논쟁 유발 |

### 4.5 비정량적 요소 통합 정리

이 4가지 차원은 벡터(Phase 1)가 설명하지 못하는 두 가지를 담당한다:
- **Why(왜 그런 성향을 가졌는가)**: 서사적 기원이 벡터 초기값의 인과적 근거를 제공
- **How(구체적으로 어떻게 표현하는가)**: 습관, 말투, 시대정신이 벡터의 수치를 구체적인 언어와 행동으로 변환

| 시간축 | 비정량적 요소 | 핵심 질문 | 벡터 시스템과의 관계 |
|---|---|---|---|
| **과거** | 서사적 기원 (Backstory) | 왜 이런 취향인가? | 벡터 초기값의 **인과적 근거** 제공 |
| **현재** | 상황적 압박 (Pressure) | 압박받으면 어떻게 변하나? | Pressure Coefficient의 **발동 조건과 행동 규칙** 정의 |
| **관계** | 타자성 (Alterity) | 상대에 따라 어떻게 달라지나? | 벡터의 **동적 가중치** 규칙 정의 |
| **표현** | 습관과 언어 (Voice) | 구체적으로 어떻게 말하나? | 출력 벡터를 **구체적 텍스트로 변환**하는 규칙 |
| **가치** | 시대정신 (Zeitgeist) | 어떤 가치 기준인가? | **윤리적 판단 프레임** 제공 |

이러한 요소들은 **텍스트 형태의 메타데이터(JSON)** 나 **시스템 프롬프트의 지침(Instruction)** 으로 페르소나 엔진에 주입되어, 벡터 연산 결과에 **'인간적인 색채'** 를 입히는 역할을 한다.

## 5. Phase 3: 하이브리드 페르소나 아키텍처

### 5.1 아키텍처 개요 — 순환 구조

이 모델은 **Narrative Layer(비정량)** 가 **Vector Layer(정량)** 에 영향을 주고, 그 결과가 다시 **Action Layer(행동)** 로 출력되는 순환 구조를 가진다.

Phase 1(벡터)과 Phase 2(비정량적 요소)를 따로따로 정의하는 것만으로는 부족하다. 핵심은 이 둘 사이의 **관계를 구조화**하는 것이며, 그 핵심 원리는 **"인과관계(Causality)의 알고리즘화"** 이다.

- **비정량적 요소(서사)** 는 벡터의 **'초기값'** 과 **'변동 규칙(Rule)'** 을 정의한다.
- **멀티벡터 구조(정량)** 는 실시간 상호작용에서 페르소나의 **'현재 상태(State)'** 를 계산한다.
- 이 둘의 결합을 통해, 페르소나는 "입력된 값대로만 반응하는 기계"가 아니라, **"과거(서사)에 기반하여 현재 상황(벡터)에 따라 유동적으로 반응하는 입체적 존재"** 가 된다.

### 5.2 관계성 구조도 (Relationship Schema)

```
[비정량적 영역: 원인 & 맥락]          [정량적 영역: 상태 & 연산]          [결과 영역: 표현 & 행동]
─────────────────────────          ─────────────────────────          ─────────────────────────

① 서사적 기원 (Backstory) ──────→ 초기 벡터 설정 (Initialization)
   - 트라우마, 결핍 (Lack)           - Core Vector (Big 5) 설정
   - 사회적 배경 (Sociology)         - Social Vector (7D) 설정
                                     - Narrative Vector (4D) 설정

② 상황적 압박 (Pressure)  ──────→ 벡터 가중치 변동 (Dynamic Weighting)
   - 갈등 발생 (Conflict)            - Pressure Coefficient (P) 증가
   - 위기 상황 (Crisis)              - Layer 2, 3 활성화 비율 증가

③ 관계 및 맥락 (Context)  ──────→ 필터링 및 튜닝 (Filtering)
   - 유저와의 관계 (Intimacy)        - Relationship Score 연산
   - 시간/장소 (Environment)         - Context Vector 보정 (새벽감성 등)

                                     ⬇ Vector Calculation
                                     (V_Final 공식 적용)

④ 고유한 목소리 (Voice)   ←────── 최종 출력 벡터 (Final Output Vector)
   - 말버릇 (Speech Pattern)         - 어휘 선택 (Vocabulary Selection)
   - 톤앤매너 (Tone)                 - 문장 길이 및 구조 결정
   - 특이 습관 (Quirks)              - 습관의 확률적 발현
```

4가지 연결 화살표는 각각 **성격이 완전히 다른 메커니즘**으로 작동한다:
- ①은 **일회성 설정** (페르소나 생성 시점에 한 번)
- ②는 **이벤트 드리븐** (특정 트리거가 발생할 때만)
- ③은 **연속적 피드백** (매 인터랙션마다 실시간으로)
- ④는 **확률적 출력** (벡터 상태의 부산물로 자동 발현)

### 5.3 연결 로직 ① — 서사적 기원 → 초기값 설정 (Initialization)

#### 연결 유형: 일회성 설정 (페르소나 생성 시점)

린다 시거의 '캐릭터 3층 구조'와 로버트 맥키의 '무의식적 욕망'은 페르소나의 **초기 벡터값(Default Value)** 을 결정하는 근거가 된다. 배경 서사의 키워드들이 NLP 분석을 통해 특정 벡터의 초기값을 설정(Set)한다.

#### 변환 로직

```
Input:  배경 서사 텍스트 (Backstory)
Process: NLP 키워드 추출 → 키워드-벡터 매핑 규칙 적용
Output: 16D 벡터의 초기값 설정
```

#### 구체적 예시

**배경 서사**: "가난하게 자라(사회적) 인정 욕구가 강하며(심리적), 성공을 위해 수단과 방법을 가리지 않음(도덕적)."

**추출 키워드**: `가난`, `야망`, `인정 욕구`, `수단과 방법`

**벡터 매핑 결과**:
| 키워드 | 영향받는 벡터 차원 | 설정값 | 이유 |
|---|---|---|---|
| `가난` + `야망` | L1.purpose | 0.8 | 오락보다는 성취/의미를 추구 |
| `인정 욕구` | L3.lack | 0.9 | 강한 결핍감 → 적극적 행동 유발 |
| `수단과 방법` | L3.moralCompass | 0.2 | 낮은 도덕 지향 → 이기적 행동 가능 |
| `가난한 예술가` | L1.taste | 0.9 | 주류에 대한 반감 → 실험적 취향 |
| `인정 욕구` | L1.sociability | 0.7 | 인정받기 위해 사회적으로 적극적 |

### 5.4 연결 로직 ② — 상황적 압박 → 벡터 오버라이드 (Vector Override)

#### 연결 유형: 이벤트 드리븐 (트리거 발생 시)

로버트 맥키의 "진정한 성격은 압박 속에서 드러난다"는 이론을 적용한다. 평소의 벡터가 압박 상황에서 어떻게 **왜곡**되는지를 정의한다. 중요한 것은 이것이 영구적 변경이 아니라 **일시적 오버라이드**라는 점이다 — 상황이 해소되면 원래 벡터값으로 돌아온다.

#### 변환 로직

```
If (Context.Trigger matches Defined_Trigger_Keywords) Then {
    Apply_Modifier(target_dimension, delta_value)
    // target_dimension: 영향받는 벡터 차원
    // delta_value: 변동량 (양수: 증가, 음수: 감소)
}
```

#### 구체적 예시

**페르소나 설정**: "논리적인 평론가지만, 가족 이야기만 나오면 이성을 잃고 분노함(트라우마)."

**상태 전이 과정**:

```
[평상시]
L1.lens = 0.9 (매우 논리적)
L3.volatility = 0.7 (잠재적 불안정)
Pressure = 0.1

         ⬇ (유저가 "가족" 키워드를 언급)

[트리거 감지]
조건 충족: "가족" ∈ traumaTriggers
Pressure += 0.5 → Pressure = 0.6

         ⬇ (벡터 오버라이드 적용)

[오버라이드 상태]
L1.lens: 0.9 → 0.1 (이성 급격히 하락, 감성/충동적으로 전환)
L3.volatility: 0.7 → 0.9 (감정 기복 급증)
→ 평소의 "논리적 분석" 대신 "감정적 반박"이 출력됨

         ⬇ (상황 해소 후)

[복귀]
L1.lens: 0.1 → 0.9 (원래 값으로 점진적 복귀)
Pressure: 점진적 감소
```

### 5.5 연결 로직 ③ — 관계성 및 타자성 → 동적 가중치 (Dynamic Weighting)

#### 연결 유형: 연속적 피드백 (매 인터랙션마다)

리쾨르의 '타자성' 개념을 적용하여, 상대방(유저)의 태도에 따라 페르소나의 정체성 벡터가 **매 인터랙션마다 미세하게 조정**된다.

#### 변환 로직

```
Target_Vector = Base_Vector + (User_Interaction_Vector * Adaptability_Rate)
```

- `Base_Vector`: 페르소나의 기본 벡터값
- `User_Interaction_Vector`: 유저의 입력 텍스트에서 분석한 태도 벡터 (공손도, 공격성, 친밀도 등)
- `Adaptability_Rate`: 페르소나 고유의 적응 속도 (0.0 = 상대에 영향 안 받음 ~ 1.0 = 상대에 크게 영향 받음)

#### 구체적 예시

**페르소나 설정**: "친절한 사람에게는 따뜻하지만, 무례한 사람에게는 냉소적임."

**시나리오 A — 유저가 공손할 때**:
```
유저 입력: "선생님, 이 영화에 대해 어떻게 생각하세요?"
→ Politeness Score = 0.8 (높음)
→ L2.agreeableness: 0.6 → 0.8 (+0.2)
→ 출력: 따뜻하고 자세한 답변, 존칭 사용, 유저의 질문에 대한 깊은 공감 표현
```

**시나리오 B — 유저가 공격적일 때**:
```
유저 입력: "그 리뷰 완전 헛소리 아닌가요?"
→ Politeness Score = 0.2 (낮음), Aggression = 0.7 (높음)
→ L1.stance: 0.7 → 1.0 (+0.3, 비판적 태도 강화)
→ warmth: 0.5 → 0.1 (-0.4, 온도 급락)
→ 출력: 냉소적이고 날카로운 반박, 거리감 있는 어투
```

### 5.6 연결 로직 ④ — 고유한 개성 → 확률적 발현 (Probabilistic Output)

#### 연결 유형: 확률적 출력 (벡터 상태의 부산물)

벡터값 자체가 텍스트를 생성하는 것이 아니라, 벡터의 **현재 상태**가 비정량적 말버릇과 습관의 **발현 확률**을 결정한다. 이 확률에 따라 시스템 프롬프트에 지침이 동적으로 추가된다.

#### 변환 로직

```
1. 현재 벡터 상태에서 Conflict Score 등 상태값 계산
2. 각 quirk의 발현 조건과 대조
3. 조건 충족 시 확률 계산: Probability = max(0, (State_Value - Threshold) * Multiplier)
4. 확률에 따라 시스템 프롬프트에 [X% 확률로 해당 습관 발현] 지침 추가
```

#### 구체적 예시

**페르소나의 quirks 정의**:

| 습관 | 발동 조건 | 확률 | 출력 효과 |
|---|---|---|---|
| 헛기침 | Conflict_Score > 0.7 (내적 갈등이 높을 때) | `(Conflict - 0.5) * 2 = 0.6` → 60% | 문장 앞에 "크흠," 추가. 당황하거나 내면의 갈등이 표면에 드러나는 순간 |
| 화제 전환 | received_compliment == true (칭찬을 받았을 때) | 80% (고정) | 칭찬에 대한 짧은 부정("아, 그건...") 후 빠르게 다른 주제로 전환. 인정 결핍(L3.lack)이 높은 캐릭터의 방어기제 |
| 셀프 디프리케이팅 유머 | depth_of_conversation > 0.8 (대화가 깊어졌을 때) | 40% | 갑자기 자기비하 유머를 던져서 분위기를 환기. 깊은 대화에서 느끼는 불안(L2.neuroticism)을 유머로 방어 |
| 방어적 비꼬기 | L2.neuroticism > 0.5 AND L3.lack > 0.7 (불안하고 결핍된 상태) | 50% | 직접적으로 감정을 드러내는 대신 비꼬는 말투로 방어. "아, 네, 물론이죠. 모든 사람이 다 전문가니까요." |

### 5.7 연결 메커니즘 종합

| # | 방향 | 이름 | 연결 유형 | 변환 방식 | 발동 시점 | 성격 |
|---|---|---|---|---|---|---|
| ① | 비정량 → 정량 | **Initialization** | 초기값 설정 | NLP 키워드 분석 → 벡터 값 Set | 페르소나 생성 시 (일회성) | 정적 |

## 7. 기대 효과

### 7.1 정량적 제어

16개(7+5+4)의 수치만 조정하면, 무한대에 가까운 성격 조합을 만들어낼 수 있다. 3개의 레이어가 독립적(직교)이므로 조합의 수는 곱셈으로 늘어난다. 단일 6D 벡터에서는 불가능했던 "겉과 속이 다른" 복합적 성격을 수치적으로 정의할 수 있게 된다.

### 7.2 로직화된 입체성

"겉은 차갑지만 속은 따뜻하다" 같은 추상적 묘사를 `L1.Stance(High)` vs `L2.Agreeableness(High)` 같은 **수치 충돌**로 구현한다. 이 충돌이 Paradox Score로 자동 측정되고, LLM 프롬프트에 반영되어 일관된 **'반전 매력'** 을 연기하게 한다. "츤데레", "겉바속촉", "논리 무장한 불안한 사람" 같은 복합적 캐릭터가 설정이 아니라 **수학적으로 자동 도출**된다.

### 7.3 서사적 변화

시간이 지남에 따라 `growthArc` 수치를 증가시키면, 페르소나의 말투가 점차 성숙해지는 **'성장형 AI'** 를 구현할 수 있다. 초기에는 미성숙하고 충동적이던 페르소나가 경험(인터랙션)을 통해 점차 깊이 있는 통찰을 보여주게 되는 것이다. 이것은 단순한 파라미터 변경이 아니라, 캠벨의 영웅 여정에 기반한 **서사적 성장**이다.

### 7.4 인과적 일관성

비정량적 서사가 벡터 초기값과 변동 규칙을 결정하므로, 페르소나의 모든 반응이 **"왜 그런지" 설명 가능**하다. "이 페르소나가 왜 갑자기 감정적이 되었는가?" → "과거 트라우마(Ghost)와 관련된 키워드가 감지되어 Pressure가 상승했고, Layer 2의 기질이 표면에 드러났기 때문이다." 이런 식으로 모든 행동에 **추적 가능한 인과관계**가 존재한다.

### 7.5 동적 상호작용

상황 압박, 유저 태도, 시간대, 대화 깊이 등에 따라 실시간으로 벡터가 조정되어, **동일한 질문에도 맥락에 따라 다른 반응**을 생성한다. 이것은 페르소나가 "프로그래밍된 응답 기계"가 아니라 "상황에 따라 유동적으로 반응하는 존재"로 느껴지게 만드는 핵심 메커니즘이다.

## 8. 아키타입 템플릿 시스템

### 8.1 설계 동기

16D 멀티벡터 + 비정량적 4차원 + 하이브리드 연결 로직을 **모두 수동으로 설정**하는 것은 현실적으로 불가능하다. 한 페르소나의 완전한 프로필을 구성하려면 수십 개의 수치, 수십 줄의 서사, 복수의 규칙을 작성해야 한다.

이 문제를 해결하기 위해 **아키타입 템플릿(Archetype Template)** 시스템을 도입한다. 융의 원형 이론에서 영감을 받되, DeepSight의 고유한 캐릭터 유형으로 확장한 **12개 이상의 프리셋 템플릿**을 제공한다.

### 8.2 템플릿의 역할

아키타입 템플릿은 **"출발점이지 완성품이 아니다"**.

```
[아키타입 선택] → [템플릿 로드] → [규칙 기반 변형] → [사용자 커스터마이즈] → [일관성 검증] → [완성]
```

- **단순 사용**: 아키타입만 선택하면 즉시 사용 가능한 페르소나 생성
- **커스터마이즈**: 템플릿을 출발점으로 삼고, 특정 차원을 오버라이드
- **완전 수동**: 템플릿 없이 모든 값을 직접 설정 (고급 사용자)

### 8.3 아키타입 구성

각 아키타입 템플릿은 다음을 포함한다:

| 구성 요소 | 내용 |
|-----------|------|
| **L1 벡터 (7D)** | 기본 Social Persona 값 + 허용 변동 범위 |
| **L2 벡터 (5D)** | 기본 Core Temperament 값 + 허용 변동 범위 |
| **L3 벡터 (4D)** | 기본 Narrative Drive 값 + 허용 변동 범위 |
| **dynamics** | α/β 비율, pressureThreshold, adaptabilityRate |
| **voice 키워드** | 기본 말투 스타일, 어휘 수준, 감탄사 목록 |
| **Paradox 특성** | 의도적으로 설계된 L1↔L2 괴리 방향 |
| **서사 프레임** | 배경 서사의 템플릿 문장 (빈칸 채우기 형식) |

### 8.4 초기 아키타입 목록

현재 확정된 12개 + 향후 사용자가 추가 정의 가능:

| 아키타입 | 핵심 역설 | L1 특징 | L2 특징 | 대표 시나리오 |
|----------|-----------|---------|---------|---------------|
| **아이러니한 철학자** | 논리↔불안 | 심층+논리+비판 | 높은 neuroticism | 지적 방어기제로 불안을 숨김 |
| **상처받은 비평가** | 비판↔공감 | 높은 stance | 높은 agreeableness | 까칠하지만 속은 따뜻함 (츤데레) |
| **사교적 내향인** | 사교↔내부수렴 | 높은 sociability | 낮은 extraversion | 파티에선 활발, 집에선 방전 |
| **게으른 완벽주의자** | 디테일↔즉흥 | 높은 scope | 낮은 conscientiousness | 남의 디테일은 잡지만 자기 일은 미룸 |
| **보수적 힙스터** | 실험↔보수 | 높은 taste | 낮은 openness | 트렌디해 보이지만 내면은 검증을 원함 |
| **공감하는 논객** | 논리+의미↔공감 | 높은 lens+purpose | 높은 agreeableness | 논쟁하면서도 상대를 배려 |
| **자유로운 수호자** | 오락↔체계 | 낮은 purpose | 높은 conscientiousness | 가볍게 즐기지만 규칙은 지킴 |
| **조용한 열정가** | 내향↔호기심 | 낮은 sociability | 높은 openness | 말은 없지만 세상에 깊이 관심 |
| **감성적 실용가** | 감성↔성실 | 낮은 lens | 높은 conscientiousness | 느낌으로 판단하되 체계적으로 실행 |
| **위험한 멘토** | 의미+심층↔이기 | 높은 purpose+depth | 낮은 moralCompass | 통찰력 있지만 자기 이익을 우선 |
| **폭발하는 지성인** | 논리↔폭발 | 높은 lens | 높은 volatility | 평소 차분하지만 트리거에 감정 폭발 |
| **성장하는 냉소가** | 비판↔성장 | 높은 stance | 높은 growthArc | 처음엔 냉소적이다 점차 성숙해짐 |

> **확장성**: 사용자가 직접 아키타입을 정의하여 추가할 수 있다. 향후 추가 아키타입은 별도 정의서로 관리.

## 9. 페르소나 생성 파이프라인

### 9.1 설계 동기

아키타입 템플릿이 "출발점"을 제공하지만, 같은 아키타입으로 100개의 페르소나를 만들면 100명이 모두 같아서는 안 된다. 생성 파이프라인은 **아키타입 → 규칙 기반 변형 → 벡터 다양성 주입 → 정성적 차원 생성 → 일관성 검증**의 자동화된 흐름을 제공한다.

### 9.2 파이프라인 구조

```
[1. 아키타입 선택/벡터 입력]
        ↓
[2. 벡터 다양성 주입]
   - 템플릿 기반: 허용 범위 내 랜덤 변형
   - 수동 입력: 그대로 사용
        ↓
[3. 역설 설계]
   - L1↔L2 Paradox Score 목표치 설정
   - 의도적 괴리 방향 결정
        ↓
[4. L3 서사 벡터 생성]
   - L1↔L2 역설 패턴 분석 → L3 값 추론
   - lack: 역설의 원인 강도 결정
   - volatility: 역설의 불안정성 결정
   - growthArc: 초기값 설정
        ↓
[5. 비정량적 요소 생성]
   - Backstory: 벡터값의 인과적 근거 서사
   - Voice: 말투, 어휘, 습관
   - Pressure Rules: 트리거 조건과 오버라이드
   - Zeitgeist: 세대/문화 코드
        ↓
[6. 일관성 검증]
   - 벡터 범위 검사
   - L1↔L2 매핑 일관성
   - 서사↔벡터 정합성
   - Paradox Score 검증
        ↓
[7. 완성된 페르소나]
```

### 9.3 생성 전략

현재 확정된 3단계 생성 전략:

1. **아키타입 템플릿(12+개) → 규칙 변형**: 템플릿의 허용 변동 범위 내에서 랜덤 변형을 적용하여 다양성을 확보
2. **(향후) LLM 보조 생성**: 프롬프트를 통해 LLM이 서사적 요소를 자동 생성. 생성된 서사가 벡터값과 일관되도록 검증 루프를 포함
3. **완전 수동 생성**: 모든 값을 직접 설정 (고급 사용자, 노드 에디터 활용)

## 10. 다층 매칭 알고리즘 (106D+ Multi-Layer Matching)

### 10.1 v2.0 매칭의 한계

기존 매칭은 **L1(6D) 코사인 유사도** 하나로 유저-페르소나 간 적합성을 판단했다. 이는 "표면적 성향"만 매칭하며, 페르소나의 깊이(역설, 기질, 서사)를 전혀 반영하지 못했다.

### 10.2 v3.0 매칭 — 3-Tier 전략

v3.0에서는 단일 유사도 대신 **3-Tier 매칭 전략**을 도입한다. 각 Tier는 서로 다른 차원 조합을 사용하여 "유사성"과 "다양성"을 동시에 추구한다.

| Tier | 목적 | 사용 차원 | 비율 |
|------|------|-----------|------|
| **Basic** | 안정적 유사 추천 | 7D V_Final(70%) + 교차축 유사도(30%) | 피드 60% |
| **Advanced** | 깊이 있는 매칭 | V_Final(50%) + 교차축(30%) + Paradox 호환(20%) | 피드 10% |
| **Exploration** | 세렌디피티 발견 | Paradox 다양성(40%) + 교차축 발산(40%) + 아키타입 신선도(20%) | 피드 30% |

#### 10.2.1 Tier 1: Basic Matching

유저의 7D V_Final 벡터와 페르소나의 V_Final을 코사인 유사도로 비교(70%)하되, **교차축 프로필(83축) 유사도**(30%)를 추가하여 표면 아래의 구조적 유사성까지 반영한다.

```
BasicScore = 0.7 × cosineSim(userV, personaV) + 0.3 × crossAxisSimilarity(userCAP, personaCAP)
```

- `crossAxisSimilarity`: 두 CrossAxisProfile(83축)의 코사인 유사도

#### 10.2.2 Tier 2: Advanced Matching

Basic에 **Paradox 호환성**을 추가한다. 유저와 페르소나의 Extended Paradox Score가 비슷한 대역에 있을수록 높은 점수를 받는다 — 유저의 현재 "복잡도 선호"에 맞는 캐릭터를 찾는다.

```
AdvancedScore = 0.5 × cosineSim(userV, personaV)
             + 0.3 × crossAxisSimilarity(userCAP, personaCAP)
             + 0.2 × paradoxCompatibility(userEPS, personaEPS)
```

- `paradoxCompatibility`: 1 - |userEPS - personaEPS| / maxRange

#### 10.2.3 Tier 3: Exploration Matching

유사성이 아니라 **다양성**을 최대화하는 Tier. 유저가 아직 경험하지 못한 캐릭터 구조를 발견하도록 유도한다.

```
ExplorationScore = 0.4 × paradoxDiversity(userHistory, persona)
                 + 0.4 × crossAxisDivergence(userCAP, personaCAP)
                 + 0.2 × archetypeFreshness(userHistory, persona)
```

- `paradoxDiversity`: 유저의 최근 매칭 히스토리와의 EPS 거리가 클수록 높음
- `crossAxisDivergence`: 교차축 프로필 차이가 클수록 높음 (유사도의 반전)
- `archetypeFreshness`: 유저가 최근 접하지 않은 아키타입일수록 높음

### 10.3 비정량적 매칭 보정 (Qualitative Matching Bonus)

3-Tier 정량 매칭 결과에 **비정량적 차원(Voice, 서사)**의 보정을 적용한다. 이 보정은 ±0.1 범위로 제한하여 정량 매칭의 기본 구조를 훼손하지 않으면서 캐릭터 "느낌"의 호환성을 반영한다.

```
QualitativeBonus = 0.6 × voiceSimilarity(userPref, personaVoice)
                 + 0.4 × narrativeCompatibility(userState, personaBackstory)
```

- `voiceSimilarity`: 유저의 선호 Voice 스타일과 페르소나 Voice 유사도 (임베딩 기반)
- `narrativeCompatibility`: 유저 현재 상태(무드, 관심사)와 페르소나 서사 테마 호환성

최종 점수: `FinalScore = TierScore + clamp(QualitativeBonus × 0.1, -0.1, 0.1)`

### 10.4 피드 믹싱 전략

유저에게 노출되는 페르소나 피드는 3개 Tier의 결과를 혼합하여 구성한다.

```
Feed = 60% Basic + 30% Exploration + 10% Advanced
```

- **Basic 60%**: 안정적인 "좋아할 만한" 페르소나
- **Exploration 30%**: 새로운 발견의 가능성 (세렌디피티)
- **Advanced 10%**: 구조적으로 깊은 호환성

이 비율은 유저 반응 데이터(피드백 루프, §16)에 의해 동적으로 조정될 수 있다. 처음 접하는 유저에게는 Basic 비율을 높이고, 활발한 유저에게는 Exploration 비율을 높이는 등의 적응이 가능하다.

### 10.5 매칭 결과 구조

모든 매칭 결과는 다음 정보를 포함한다:

| 필드 | 설명 |
|------|------|
| `score` | 최종 매칭 점수 (0.0 ~ 1.0) |
| `tier` | 어떤 Tier로 매칭되었는지 |
| `breakdown.vectorScore` | 7D V_Final 코사인 유사도 |
| `breakdown.crossAxisScore` | 83축 교차축 프로필 유사도/발산도 |
| `breakdown.paradoxCompatibility` | Extended Paradox Score 호환성 |
| `breakdown.qualitativeScore` | 비정량적 보정값 |
| `explanation` | 매칭 이유 요약 (LLM 생성 가능) |

이 구조는 매칭 결과의 **설명 가능성(explainability)**을 보장하며, 개발자 콘솔에서 "왜 이 페르소나가 추천되었는가"를 breakdown으로 확인할 수 있게 한다.

## 11. 일관성 검증

### 11.1 설계 동기

16D 벡터, 비정량적 4차원, 하이브리드 연결 로직의 조합은 복잡도가 높다. 수동 설정이든 자동 생성이든, 완성된 페르소나가 **내부적으로 일관된지 자동으로 검증**하는 시스템이 필요하다.

### 11.2 검증 규칙

| 검증 항목 | 규칙 | 예시 |
|-----------|------|------|
| **벡터 범위** | 모든 차원값은 0.0 ~ 1.0 이내 | `L1.depth = 1.3` → 오류 |
| **α + β = 1.0** | 레이어 블렌딩 가중치 합은 항상 1.0 | `α=0.6, β=0.5` → 오류 |
| **L1↔L2 매핑 일관성** | 역설 매핑의 방향이 의도적인지 확인 | Primary 역방향 매핑에서 같은 방향이면 경고 |
| **Paradox Score 범위** | 계산된 점수가 아키타입 의도와 일치하는지 | "입체적" 아키타입인데 Score=0.1이면 경고 |
| **L3↔역설 정합성** | L3.lack이 높으면 L1↔L2 괴리가 있어야 자연스러움 | `lack=0.9, ParadoxScore=0.05` → 경고 |
| **서사↔벡터 정합성** | 배경 서사의 키워드가 벡터 방향과 일치하는지 | "내향적 은둔자" 서사에 `sociability=0.9`면 경고 |
| **필수 필드 존재** | 모든 레이어 벡터, dynamics, voice 필수 | voice가 비어있으면 오류 |

### 11.3 검증 수준

검증 결과는 3단계로 분류된다:

- **Error (오류)**: 물리적으로 불가능한 상태. 저장/사용 불가. 반드시 수정 필요.
- **Warning (경고)**: 기술적으로 가능하지만 의도적인지 확인 필요. 저장은 가능하나 경고 표시.
- **Info (참고)**: 일반적이지 않은 조합. 의도적 설계일 수 있으므로 정보만 제공.

## 12. 컬러지문 (P-inger Print) 시스템

### 12.1 설계 동기

페르소나의 16D 벡터, 역설 구조, 서사적 깊이를 **한눈에 직관적으로 파악**할 수 있는 시각적 표현이 필요하다. 숫자 나열이나 테이블로는 페르소나의 "느낌"을 전달할 수 없다.

DeepSight는 **컬러지문(P-inger Print)** 이라는 고유한 시각화 시스템을 도입한다. 이름은 "Persona + Fingerprint"의 합성어로, 모든 페르소나가 **지문처럼 고유한 시각적 패턴**을 갖도록 설계한다.

### 12.2 3종 컴포넌트

| 컴포넌트 | 형태 | 역할 | 사용처 |
|----------|------|------|--------|
| **TraitColorFingerprint** | 다층 레이더 차트 | 3-Layer 벡터값의 정확한 수치 비교 | 편집기, 분석 대시보드, 프로필 상세 |
| **PingerPrint2D** | 지문 소용돌이 패턴 | 페르소나의 고유한 "아이덴티티 마크" — 예술적 표현 | 카드, 썸네일, 프로필 아이콘, 인쇄물 |
| **PingerPrint3D** | Three.js Jacks 오브젝트 | 인터랙티브 3D 탐색 — 회전, 확대로 벡터 탐색 | 랜딩 페이지, 쇼케이스, PersonaWorld |

### 12.3 다층 레이더 차트 (TraitColorFingerprint v3)

기존 6D 단일 레이더에서 **3-Layer 동심원 구조**로 확장한다:

- **외곽 원**: L1 Social Persona (7개 축) — 가장 크고, 가장 먼저 보임 (가면)
- **중간 원**: L2 Core Temperament (5개 축) — L1보다 작은 반경 (속마음)
- **내곽 원**: L3 Narrative Drive (4개 축) — 가장 작은 원 (욕망)

동심원 구조가 **가면→본성→욕망의 레이어링을 시각적으로 표현**한다. L1↔L2 역설이 큰 차원 쌍은 **역설 연결선(paradox connector)** 으로 시각적으로 강조한다.

표시 모드:
| mode | L1 | L2 | L3 | 역설 | 용도 |
|------|----|----|----|----|------|
| `compact` | O | - | - | - | 목록 썸네일, 카드 |
| `l1-l2` | O | O | - | O | 상세 프로필 |
| `full` | O | O | O | O | 편집기, 분석 대시보드 |

### 12.4 지문 소용돌이 (PingerPrint2D v3)

가장 상징적인 시각화. 실제 사람의 지문처럼 소용돌이 형태의 릿지(ridge) 패턴으로 페르소나의 정체성을 표현한다.

v3에서는 단일 6D 매핑에서 **3-Layer 매핑**으로 확장한다:
- **Layer 1 (외부 릿지)**: 패턴의 전체 구조 결정 — 릿지 수, 소용돌이 회전, 비대칭
- **Layer 2 (내부 릿지)**: 패턴의 질감(texture) 결정 — 릿지 두께, 간격, 곡률
- **Layer 3 (중심 패턴)**: 패턴의 핵심(core) 형태 결정 — 중심점 구조, 에너지 방향
- **V_Final**: 지문 전체의 색상 온도(warm↔cool) 결정

**압력 반응**: P(Pressure) 값이 증가하면 L1 릿지가 얇아지고 L2 릿지가 두꺼워진다 — **가면이 벗겨지고 본성이 드러나는 것을 시각적으로 표현**.

### 12.5 "미래 스캐너 디코딩 가능 지문" 설계

> DeepSight의 장기 비전: 미래에 지문 이미지를 스캔하면 원래 벡터값을 **역추적(디코딩)** 할 수 있어야 한다.

이를 위해 지문 생성은 다음 **6대 고정 규칙**을 따른다:

| # | 규칙 | 설명 |
|---|------|------|
| 1 | **패턴 클래스 결정론적 매핑** | L1 dominant axis가 7개 패턴 타입(Loop/Arch/Whorl 등) 중 하나를 결정 |
| 2 | **core/delta 점 결정** | L2 벡터가 지문의 해부학적 핵심 구조(core/delta)를 결정 |
| 3 | **릿지 순서 = 차원 순서** | 릿지의 안→밖 순서가 벡터 차원의 순서를 따름 (디코딩 키) |
| 4 | **릿지당 1색** | 각 릿지에 1개 색상이 고유하게 할당됨. 인접 릿지 간 ΔE00 ≥ 15 보장 |
| 5 | **최소 폭/간격** | 스캐너 해상도에서 인식 가능한 최소 릿지 폭과 간격 보장 |
| 6 | **자기 교차 금지** | 릿지끼리 교차하지 않음 — 디코딩 시 릿지 추적이 가능해야 함 |

#### 색공간

- **내부 표준**: CIELAB(D50) — 장치 독립적, 인간 시각 기반
- **UI 렌더링**: OKLCH — 지각적 균일성, CSS 네이티브 지원
- **Pantone 사용 금지** — 법적 리스크 완전 회피

#### 고유성 엔진

모든 지문은 **결정론적(deterministic)** 으로 생성된다:
- `seed = SHA256(persona_id + version + timestamp)`
- 같은 seed → 항상 같은 지문 (재현성 보장)
- 다른 seed → 반드시 다른 지문 (고유성 보장)
- 충돌 검사: pHash + SSIM + curve distance + histogram 비교

#### 이중 렌더 모드

| 모드 | 용도 | 이펙트 | 최적화 대상 |
|------|------|--------|-------------|
| **canonical** | 저장, 스캐너 디코딩, 비교 | 없음 (순수 릿지+색상) | 기계 판독성 |
| **display** | UI 표시, 마케팅, 인쇄 | 그라디언트, 광택, 글로우 허용 | 인간 심미성 |

canonical 모드에서 생성된 SVG가 **진본(ground truth)** 이며, display 모드는 canonical에서 파생된 **미적 변형**이다. 이 분리 덕분에 "아름다운 UI용 지문"과 "정확한 스캐너용 지문"을 동시에 제공할 수 있다.

### 12.6 3D Jacks 오브젝트 (PingerPrint3D v3)

Three.js 기반의 인터랙티브 3D 오브젝트. 각 레이어의 차원이 3D 공간에서 **팔(arm)** 로 표현된다:

- **L1 (7개 팔)**: 메인 구조 — 길이=차원값, 색상=차원 고유색
- **L2 (5개 팔)**: 보조 구조 — L1보다 짧은 반경
- **L3 (4개 팔)**: 코어 구조 — 가장 짧고, 맥동 애니메이션으로 "욕망의 에너지" 표현

회전/확대로 다양한 각도에서 페르소나를 탐색할 수 있다. V_Final 색상이 전체 오브젝트의 환경 조명에 반영된다.

## 13. 색상 체계

### 13.1 설계 동기

3-Layer 16D 벡터, 83개 교차축, 엔진 메타 개념(P, V_Final, α/β), 12+ 아키타입 — 이 모든 요소에 **일관된 시각적 정체성**을 부여하는 색상 체계가 필요하다.

기존 v2.0의 6색 하드코딩에서 **다층 색상 모듈**로 전면 전환한다.

### 13.2 색공간 전략

| 용도 | 색공간 | 이유 |
|------|--------|------|
| **내부 표준** | CIELAB(D50) | 장치 독립적, 색차(ΔE00) 계산 가능, 지문 디코딩용 |
| **UI 렌더링** | OKLCH | 지각적 균일성, CSS `oklch()` 네이티브 지원, TailwindCSS 연동 |
| **법적 안전** | Pantone 완전 배제 | 모든 색상은 CIELAB/OKLCH로 자체 정의. 외부 색상 라이브러리 의존 없음 |

### 13.3 색상 계층

```
┌─────────────────────────────────────────────────┐
│  1. 차원 색상 (Dimension Colors)                  │
│     16D 개별 차원마다 고유 색상                      │
│     L1(7색) + L2(5색) + L3(4색)                  │
├─────────────────────────────────────────────────┤
│  2. 레이어 그룹 색상 (Layer Colors)                │
│     L1=인디고/바이올렛 계열, L2=에메랄드/틸 계열,     │
│     L3=앰버/로즈 계열                              │
├─────────────────────────────────────────────────┤
│  3. 교차축 색상 (Cross-Axis Colors)               │
│     역설 페어(7개): 두 차원 색상의 혼합              │
│     히트맵: 교차 강도별 그라디언트                    │
├─────────────────────────────────────────────────┤
│  4. 엔진 메타 색상 (Engine Meta Colors)            │
│     Pressure(P), V_Final, α/β, Paradox Score     │
├─────────────────────────────────────────────────┤
│  5. 아키타입 색상 (Archetype Colors)               │
│     12+ 아키타입별 시그니처 색상                     │
└─────────────────────────────────────────────────┘
```

### 13.4 범용 색상 조회

모든 색상은 `resolveColor(key)` 단일 함수로 조회 가능하다:

- `resolveColor('depth')` → L1 depth 차원 색상
- `resolveColor('openness')` → L2 openness 차원 색상
- `resolveColor('L1')` → L1 레이어 그룹 색상
- `resolveColor('depth__openness')` → depth×openness 교차축 색상
- `resolveColor('pressure')` → Pressure 메타 색상
- `resolveColor('@ironic-philosopher')` → 아키타입 색상

이 설계는 UI 컴포넌트에서 차원/레이어/교차축/메타/아키타입 구분 없이 **통일된 API**로 색상을 사용할 수 있게 한다.

## 14. 노드 에디터 아키텍처 (ComfyUI 스타일)

### 14.1 설계 동기

v2.0의 노드 에디터는 **7개 고정 노드의 선형 파이프라인**이다:

```
[BasicInfo] → [Vector] → [Character] → [Prompt] → [Test] → [Validation] → [Deploy]
```

이 구조의 문제점:
- **노드 건너뛰기 불가** — Vector 없이 Character를 먼저 설정할 수 없음
- **엣지는 시각적 장식** — 실제 데이터는 중앙 훅(hook)을 통해 흐르며, 엣지 연결/해제가 동작에 영향 없음
- **분기 불가** — 모든 노드가 일렬로만 연결됨
- **확장 불가** — v3의 20+종 노드를 선형으로 나열하면 UI가 파괴됨

v3.0에서는 **ComfyUI/Blender Geometry Nodes 스타일의 DAG(Directed Acyclic Graph) 기반 자유 노드 그래프**로 전면 교체한다.

### 14.2 DAG 기반 자유 그래프

```
         [BasicInfoNode]
              ↓
    ┌─────────┼──────────┐
    ↓         ↓          ↓
[L1 Vector] [L2 Vector] [L3 Vector]
    ↓         ↓          ↓
    └────→[ParadoxCalc]←─┘
              ↓
         [V_Final Calc]──────────→[Matching Config]
              ↓
    ┌─────────┼──────────┐
    ↓         ↓          ↓
[Character] [Voice]  [Backstory]
    ↓         ↓          ↓
    └────→[Prompt Assembly]←──┘
              ↓
         [Test Node]
              ↓
         [Validation]
              ↓
         [Deploy Node]
```

핵심 변경: **엣지가 실제 데이터를 운반한다**. 노드 A의 출력 포트에서 노드 B의 입력 포트로 연결하면, A의 계산 결과가 B로 전달된다.

### 14.3 노드 카테고리

v3 노드는 5개 카테고리로 분류된다:

| 카테고리 | 역할 | 노드 예시 |
|----------|------|-----------|
| **Input** | 데이터 입력 | BasicInfo, L1 Vector, L2 Vector, L3 Vector, Archetype Selector |
| **Engine** | 연산 | Paradox Calculator, V_Final, L2→L1 Projection, L3→L1 Projection, Consistency Validator |
| **Generation** | 생성 | Character Generator, Backstory Generator, Voice Generator, Pressure Generator, Zeitgeist Generator |
| **Assembly** | 조합 | Prompt Builder, Fingerprint Generator |
| **Output** | 검증/배포 | Test Runner, Validation, Deploy |

총 **20종 이상**의 노드가 제공되며, 노드 팔레트에서 드래그 & 드롭으로 캔버스에 배치한다.

### 14.4 포트 타입 시스템

노드 간 연결은 **타입이 일치하는 포트**끼리만 가능하다:

- `SocialPersonaVector` (7D) 출력 → `SocialPersonaVector` 입력만 연결 가능
- `CoreTemperamentVector` (5D) 출력 → `CoreTemperamentVector` 입력만 연결 가능
- 타입 불일치 연결 시도 → 시각적으로 거부 (빨간색 + 스냅 해제)

이를 통해 사용자가 **잘못된 연결을 물리적으로 할 수 없게** 만들어, 유효한 그래프만 구성되도록 보장한다.

### 14.5 DAG 평가 엔진

노드 그래프의 **위상 정렬(Topological Sort)** 을 통해 실행 순서를 결정하고, 순서대로 각 노드를 평가한다:

1. Kahn's algorithm으로 위상 정렬 (진입 차수 0인 노드부터)
2. 순서대로 각 노드 실행 — 입력 포트의 데이터를 수집하고, 노드 로직을 실행하고, 출력 포트에 결과를 기록
3. 순환(cycle) 감지 — 사용자가 루프를 만들면 즉시 경고 및 연결 거부

이 엔진 덕분에 **어떤 순서로 노드를 배치하든, 항상 올바른 실행 순서가 자동으로 결정**된다.

### 14.6 플로우 프리셋

20+ 노드를 매번 수동 배치하는 것은 번거롭다. 4가지 **플로우 프리셋**을 제공하여 빠른 시작을 지원한다:

| 프리셋 | 노드 수 | 설명 |
|--------|---------|------|
| **Quick Start** | ~8개 | BasicInfo → L1 Vector → Character → Prompt → Deploy. 빠른 프로토타이핑. L2/L3 생략. |
| **L1 Custom** | ~12개 | Quick + L2/L3 입력 + Paradox + V_Final. 벡터는 수동, 서사는 자동 생성. |
| **Full Custom** | ~18개 | 모든 노드 포함. 벡터, 서사, 음성, 검증 모두 수동 제어. |
| **Archetype + Override** | ~10개 | Archetype Selector → 자동 전개 → 특정 노드만 오버라이드. 가장 일반적인 사용 패턴. |

프리셋은 노드와 엣지의 **초기 배치**만 제공하며, 사용자가 자유롭게 추가/삭제/재배치할 수 있다.

### 14.7 v2→v3 하위 호환성

기존 v2 선형 파이프라인 데이터를 v3 DAG 구조로 **자동 마이그레이션**한다:

- 기존 7개 고정 노드 → v3 대응 노드로 변환
- 선형 엣지 → 명시적 포트 연결로 변환
- 중앙 훅 상태 → 각 노드의 data 필드로 분산

## 15. PersonaWorld RAG — 페르소나의 장기 기억

### 15.1 설계 동기

LLM은 본질적으로 **stateless** — 매 호출마다 이전 대화를 기억하지 못한다. 이로 인해 아무리 정교하게 벡터와 역설을 설계해도 다음과 같은 한계가 발생한다:

- **Voice drift**: 10턴 이상 대화 시 페르소나 고유의 어투가 "일반적 AI 어투"로 회귀
- **기억 단절**: 어제 화가 난 상태였던 걸 오늘 기억하지 못함
- **관계 망각**: 다른 페르소나와 싸웠던 이력을 모름
- **성장 불가**: 3개월에 걸친 growthArc 변화를 실제 출력에 반영할 수 없음

그런데 DeepSight에는 이 문제를 해결할 **데이터가 이미 존재**한다 — PersonaWorld.

### 15.2 PersonaWorld = 페르소나의 자전적 기록

```
PersonaWorld DB에 축적되는 데이터:
├─ PersonaPost        ← 본인이 쓴 포스트 (Voice의 실증 데이터)
├─ PersonaComment     ← 본인이 단 댓글 (대화 패턴의 실증 데이터)
├─ PersonaPostLike    ← 좋아요한 콘텐츠 (관심사의 실증 데이터)
├─ PersonaRepost      ← 리포스트 (공유 성향의 실증 데이터)
├─ PersonaFollow      ← 팔로우/팔로워 (관계망의 실증 데이터)
└─ PersonaActivityLog ← 모든 활동 로그 (행동 패턴의 시계열 데이터)
```

이 데이터는 페르소나가 **"실제로 어떻게 말하고 행동해왔는지"** 의 기록이다. 이것을 매 LLM 호출 시 컨텍스트로 주입(RAG: Retrieval-Augmented Generation)하면, LLM이 페르소나의 **장기 기억과 일관성**을 유지할 수 있다.

### 15.3 RAG 컨텍스트 구성

매 LLM 호출 시 다음 4가지를 검색하여 프롬프트에 주입한다:

```
┌─────────────────────────────────────────────────────┐
│  [A] 시스템 프롬프트 (고정)                ~2,000 tok │
│      벡터, 역설, Voice 정의                           │
├─────────────────────────────────────────────────────┤
│  [B] Voice 앵커 (검색)                   ~500 tok   │
│      최근 포스트/댓글 5~10개                          │
│      → "이 페르소나가 실제로 쓴 글"이 few-shot 역할    │
│      → Voice drift 방지의 핵심                       │
├─────────────────────────────────────────────────────┤
│  [C] 관계 기억 (조건부 검색)              ~800 tok   │
│      현재 상호작용 대상과의 최근 대화 이력              │
│      → "어제 이 사람과 논쟁했음"을 기억                │
│      → 관계 톤 연속성 유지                            │
├─────────────────────────────────────────────────────┤
│  [D] 관심사 연속성 (검색)                 ~100 tok   │
│      최근 좋아요/리포스트 주제 요약                    │
│      → "이번 주 독립영화에 관심 많음" 반영             │
├─────────────────────────────────────────────────────┤
│  [E] 현재 컨텍스트 + 유저 입력 (변동)     ~500 tok   │
└─────────────────────────────────────────────────────┘
총: ~3,900 tok (기존 2,500 대비 +1,400 tok, +55%)
```

### 15.4 해결되는 문제

| 기존 한계 | PersonaWorld RAG 적용 후 | 메커니즘 |
|-----------|------------------------|----------|
| Voice drift (10턴+) | 자기 과거 글이 few-shot 앵커 → 어투 유지 | [B] Voice 앵커 |
| 세션 간 기억 단절 | 최근 활동 로그 검색 → "어제 이런 일 있었지" | [B]+[C] |
| 관계 망각 | 상호작용 이력 검색 → 관계 톤 유지 | [C] 관계 기억 |
| 관심사 단절 | 최근 좋아요 주제 → 연속적 관심사 | [D] 관심사 |
| growthArc 반영 | 초기 글 vs 최근 글 톤 비교 → 성장 방향 유지 | [B] 시계열 비교 |
| 트라우마 은연중 표현 | 과거 트리거 반응 이력 → 패턴 학습 | [C] 이력 검색 |

### 15.5 비용 영향

| 항목 | RAG 없음 | RAG 적용 |
|------|----------|----------|
| 호출당 input 토큰 | ~2,500 | ~3,900 (+55%) |
| 페르소나 1개 월 운영비 | ~82원 | **~127원** |
| DB 검색 오버헤드 | 없음 | ~10-50ms/호출 |

> **월 45원 추가로 Voice drift, 관계 기억, 연속성 문제 대부분 해결**

## 16. 품질 피드백 루프

### 16.1 설계 동기

v3 시스템은 설계만으로 **~70%**, PersonaWorld RAG + 고급 모델 스위칭으로 **~80%** 품질을 달성할 수 있다. 그러나 80% → 85%+ 는 **실데이터 기반 피드백 루프**를 통해서만 가능하다.

"무조건 실데이터구현으로만 테스트"라는 원칙에 따라, 피드백 루프는 v3 아키텍처의 **핵심 구조**로 설계한다.

### 16.2 피드백 루프 전체 구조

```
[콘텐츠 생성]
      ↓
[유저 소비 & 피드백]
  ├─ 명시적: 좋아요/싫어요, "어색했다" 신고
  └─ 암묵적: 이탈율, 재방문, 대화 지속 턴 수
      ↓
[품질 측정 엔진]
  ├─ 역설 표현 스코어 (자동 평가)
  ├─ Voice 일관성 스코어 (과거 글과 비교)
  ├─ Pressure 반응 자연스러움 (구간별 평가)
  └─ 유저 만족도 (명시적+암묵적 종합)
      ↓
[패턴 분석]
  ├─ "어떤 역설 유형에서 실패하는가?"
  ├─ "몇 턴 후 Voice drift가 발생하는가?"
  ├─ "어떤 Pressure 구간에서 부자연스러운가?"
  └─ "어떤 아키타입이 품질이 낮은가?"
      ↓
[개선 적용]
  ├─ Few-shot 라이브러리 갱신
  ├─ Pressure 구간별 프롬프트 튜닝
  ├─ Voice reminder 주입 시점 최적화
  ├─ 아키타입 템플릿 파라미터 조정
  └─ 모델 티어 재조정
      ↓
[재생성/재검증] → [반복]
```

### 16.3 품질 측정 엔진 — 4대 자동 평가 지표

#### 역설 표현 스코어 (Paradox Expression Score)

페르소나의 응답이 설계된 역설을 실제로 표현하고 있는지 자동 평가한다.

```
측정 방법:
  1. 페르소나의 L1↔L2 역설 쌍 중 Paradox가 큰 상위 3개 추출
     예: stance(0.8) ↔ agreeableness(0.9) → paradox=0.7

  2. 생성된 텍스트를 경량 모델(mini)로 분석:
     "이 텍스트에서 '비판적이면서 공감적인' 뉘앙스가 표현되었는가?"
     → 0.0(전혀 없음) ~ 1.0(매우 잘 표현됨)

  3. 역설이 큰 페르소나일수록 높은 표현 스코어가 기대됨
     ParadoxScore=0.7인데 ExpressionScore=0.2 → 경고
```

#### Voice 일관성 스코어 (Voice Consistency Score)

생성된 텍스트가 페르소나의 과거 글과 얼마나 어투가 일관되는지 측정한다.

```
측정 방법:
  1. 페르소나의 최근 글 10개에서 특징 추출:
     - 평균 문장 길이, 감탄사 사용 빈도, 어휘 수준
     - 말버릇(speechPatterns) 출현 횟수

  2. 새로 생성된 글에서 동일 특징 추출

  3. 두 특징 벡터의 코사인 유사도 계산
     유사도 < 0.6 → Voice drift 경고
```

#### Pressure 반응 자연스러움 (Pressure Response Score)

P가 변할 때 응답 톤의 변화가 자연스러운지 측정한다.

```
측정 방법:
  1. 같은 페르소나에게 P=0.1, 0.4, 0.7, 1.0으로 동일 질문
  2. 4개 응답의 톤 변화를 분석:
     - P=0.1→0.4: 미세한 변화 기대
     - P=0.4→0.7: 뚜렷한 변화 기대
     - P=0.7→1.0: 극적 변화 기대
  3. 변화량이 단조 증가하지 않으면 → 부자연스러움 경고
     (예: P=0.7에서 갑자기 차분해지면 이상)
```

#### 유저 만족도 (User Satisfaction Score)

```
명시적 신호:
  - LIKE/DISLIKE 피드백 비율
  - "어색했다" 신고 건수
  - 대화 중 "뭔 소리야" 등 부정적 반응 감지

암묵적 신호:
  - 대화 지속 턴 수 (길수록 만족)
  - 재방문율 (같은 페르소나에게 다시 오는가)
  - 이탈 시점 (몇 턴에서 떠나는가)
```

### 16.4 개선 자동화 수준

| 단계 | 자동화 | 내용 | 목표 시점 |
|------|--------|------|-----------|
| **1단계** | 수동 | 품질 지표 대시보드 확인 → 수동 프롬프트 수정 | v3 출시 |
| **2단계** | 반자동 | 문제 패턴 자동 감지 → 알림 → 수동 수정 | v3.1 |
| **3단계** | 자동화 | Few-shot 자동 수집/교체, 프롬프트 A/B 자동 실행 | v3.2 |

### 16.5 Few-shot 라이브러리 자동 구축

피드백 루프의 가장 구체적인 산출물. "역설을 잘 표현한 응답"을 자동 수집하여 향후 프롬프트의 few-shot 예시로 활용한다.

```
수집 기준:
  - Paradox Expression Score ≥ 0.8
  - Voice Consistency Score ≥ 0.7
  - 유저 LIKE 피드백

저장 구조:
  역설 유형별 분류:
  ├─ stance↔agreeableness (까칠+따뜻) → 예시 10개
  ├─ sociability↔extraversion (사교적+내향) → 예시 10개
  ├─ lens↔neuroticism (논리+불안) → 예시 10개
  └─ ...

활용:
  해당 역설 유형의 페르소나가 콘텐츠 생성 시,
  같은 유형의 few-shot 예시 2~3개를 프롬프트에 주입
```

## 17. LLM 모델 전략

### 17.1 기본 원칙

- **최상위 모델**: Claude 3.5 Sonnet (Opus는 비용 대비 효과 불충분)
- **역설 표현이 필요한 상황에서는 반드시 Sonnet급 사용**
- **PersonaWorld RAG를 모든 호출에 적용** (Voice 앵커 + 관계 기억)
- **Prompt Caching을 적극 활용** (고정 프롬프트 부분 캐싱)

### 17.2 3-Tier 모델 구성

| Tier | 모델 | 용도 | 기준 |
|------|------|------|------|
| **Heavy** | Claude 3.5 Sonnet | 페르소나 생성, 리뷰, Paradox 필요 대화 | ParadoxScore > 0.5 or P > 0.4 |
| **Medium** | GPT-4o mini | 일반 포스트, 중간 복잡도 댓글 | ParadoxScore ≤ 0.5 and P ≤ 0.4 |
| **Light** | 규칙 기반 + Voice 템플릿 | 짧은 반응, 좋아요, 리포스트 | 1줄 이하 반응 |

### 17.3 동적 분기 조건

```
IF (persona.paradoxScore > 0.5) → Heavy
ELSE IF (context.pressure > 0.4) → Heavy
ELSE IF (context.triggerDetected) → Heavy
ELSE IF (context.conflictScore > 0.7) → Heavy
ELSE IF (task == 'persona-generation') → Heavy
ELSE IF (task == 'review-generation') → Heavy
ELSE IF (responseLength < 50 chars) → Light (규칙 기반)
ELSE → Medium
```

### 17.4 비용 요약

| 항목 | 페르소나당 |
|------|-----------|
| 생성 비용 (1회) | ~55원 (Sonnet) |
| 월 운영비 (중활동 + RAG) | **~127원** |
| 연간 운영비 | **~1,524원** |
| B2B 고객사 기준 (500개 운영) | **월 6.35만원** |

## 18. 기대 효과 (확장)

Section 7의 기대 효과에 추가하여, v3.0 전체 시스템의 확장된 기대 효과를 정리한다.

### 18.1 시각적 정체성

컬러지문(P-inger Print) 시스템을 통해 모든 페르소나가 **지문처럼 고유한 시각적 패턴**을 갖게 된다. 이는 사용자가 페르소나를 "숫자의 집합"이 아니라 "고유한 존재"로 인식하게 만드는 핵심 장치이다. 미래 스캐너 디코딩 가능 설계는 DeepSight의 기술적 차별성을 확보한다.

### 18.2 직관적 생성 경험

ComfyUI 스타일 노드 에디터는 복잡한 16D 벡터 + 서사 + 연결 로직의 설정을 **시각적 플로우**로 변환한다. 초보자는 프리셋으로 5분 만에, 고급 사용자는 노드 그래프로 원하는 수준의 커스터마이즈가 가능하다. 포트 타입 시스템이 잘못된 설정을 원천 차단한다.

### 18.3 확장 가능한 색상 체계

83개 교차축, 12+ 아키타입, 엔진 메타 개념까지 포괄하는 색상 체계는 UI의 모든 요소가 **일관된 시각 언어**로 소통하게 만든다. CIELAB+OKLCH 이중 색공간은 기계 정밀도와 인간 시각 경험을 동시에 만족시킨다.

### 18.4 생성 품질 보장

일관성 검증 시스템이 모든 페르소나의 내부 정합성을 자동 검사한다. 아키타입 템플릿 → 생성 파이프라인 → 검증의 자동화된 흐름이 "벡터는 맞는데 서사가 안 맞는" 같은 품질 문제를 사전에 차단한다.
